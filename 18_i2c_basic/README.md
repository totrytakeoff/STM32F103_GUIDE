# 第 18 课：I2C 基础

## 1. 本课要学什么

前面你已经学过两种常见通信接口：

- `UART` — 异步，2 线，点对点
- `SPI` — 同步，4 线，一主多从

这一课进入第三种最常见的板级通信总线：**I2C**

I2C 的特点：

- 只有两根线：`SCL`（时钟）和 `SDA`（数据）
- 总线上的设备共享这两根线，靠地址区分
- 引脚是开漏输出，需要**外部上拉电阻**
- 支持多主机、多从机

### 三种通信接口对比

| 特性 | UART | SPI | I2C |
|------|------|-----|-----|
| 线数 | 2（TX/RX） | 4（SCK/MOSI/MISO/NSS） | **2（SCL/SDA）** |
| 通信方式 | 异步 | 同步 | **同步** |
| 设备寻址 | 无 | 通过 NSS 片选 | **通过 7 位地址** |
| 电气特性 | 推挽 | 推挽 | **开漏 + 上拉** |
| 多从机 | 否 | 是（额外 NSS 线） | **是（共线寻址）** |
| 速度 | 通常 ≤115200bps | 可达 18MHz+ | 100kHz~3.4MHz |

## 2. 本课最终 Demo

### 2.1 使用器件

本课选用最经典、最适合 I2C 入门的外设：**AT24C02 EEPROM**

- 容量 256 字节（2Kbit）
- I2C 接口，7 位地址 0x50（A0/A1/A2 接地时）
- 协议简单，读写逻辑清晰

### 2.2 硬件连接

I2C1 默认引脚：

- `PB6` → `I2C1_SCL`
- `PB7` → `I2C1_SDA`

AT24C02 连接：

| AT24C02 引脚 | 接至 |
|-------------|------|
| VCC | 3.3V |
| GND | GND |
| SCL | PB6 |
| SDA | PB7 |
| WP | GND（写保护关） |
| A0/A1/A2 | GND（地址全 0） |

**非常重要的额外硬件：**

- SCL 和 SDA 必须通过上拉电阻（典型 4.7kΩ）接到 3.3V
- 很多 EEPROM 成品模块已自带上拉电阻

### 2.3 运行现象

程序每隔约 1 秒：

1. 往 EEPROM 地址 0x00 写一个字节
2. 等待 10ms（EEPROM 内部写周期）
3. 从同一地址读回
4. 比较读写结果
5. 一致 → LED 翻转；不一致 → LED 常亮（错误）

## 3. I2C 关键概念

### 3.1 开漏 + 上拉电阻

这是 I2C 与 UART/SPI 最根本的电气差异。

**开漏输出**：设备只能把线拉低，不能主动拉高。
**上拉电阻**：当所有设备都不拉低时，电阻把总线拉回高电平。

为什么这样设计？因为 I2C 是多主总线，多个设备共享 SCL 和 SDA。如果使用推挽输出，两个设备同时输出相反电平会短路。

上拉电阻典型值：4.7kΩ（100kHz 标准模式）或 2.2kΩ（400kHz 快速模式）。

> **I2C 总线没有上拉，通信大概率不成立。**

### 3.2 起始位（START）

```
SCL = 1, SDA = 1（空闲）→ SDA 从高变低（SCL 仍为高）
```

标志一次通信开始。

### 3.3 停止位（STOP）

```
SDA 从低变高（SCL 为高）
```

标志一次通信结束。

### 3.4 应答位（ACK/NACK）

每 8 位数据后，接收方在第 9 个时钟周期回复：

- ACK（SDA 被拉低）："收到了，继续"
- NACK（SDA 保持高）："没收到或不想继续"

### 3.5 重复起始（Repeated Start / RESTART）

在**不产生 STOP** 的情况下再次产生 START。

用于切换通信方向（如 EEPROM 随机读：先写地址 → 再切到读方向）。

### 3.6 7 位地址 vs 8 位地址字节

这是 I2C 最经典的混乱点。

| 概念 | 值 | 说明 |
|------|-----|------|
| 7 位器件地址 | 0x50 | AT24C02 的固有地址 |
| 写地址字节 | 0xA0 = 0x50<<1 \| 0 | 地址左移 1 位 + R/W=0 |
| 读地址字节 | 0xA1 = 0x50<<1 \| 1 | 地址左移 1 位 + R/W=1 |

寄存器版中需要手动拼接 R/W 位；HAL 版传入 0xA0，HAL 自动处理。

## 4. 寄存器版核心寄存器

### 4.1 `I2C1->CR2.FREQ`

APB1 时钟频率（MHz）。本课 PCLK1=36MHz，所以 FREQ=36。

**FREQ 配置错误，I2C 时序全错。**

### 4.2 `I2C1->CCR`

时钟控制寄存器，决定 SCL 频率。

标准模式公式：`CCR = PCLK1 / (2 × Fscl) = 36MHz / (2 × 100kHz) = 180`

### 4.3 `I2C1->TRISE`

最大上升时间寄存器。

标准模式：`TRISE = PCLK1(MHz) + 1 = 36 + 1 = 37`

### 4.4 `I2C1->CR1`

| 位 | 字段 | 本课值 | 含义 |
|----|------|--------|------|
| bit 0 | PE | 1 | I2C 总使能 |
| bit 8 | START | 软件控制 | 产生 START 条件 |
| bit 9 | STOP | 软件控制 | 产生 STOP 条件 |
| bit 10 | ACK | 1 | 接收时应答 |

### 4.5 `I2C1->SR1`（状态寄存器 1）

| 位 | 标志 | 含义 |
|----|------|------|
| bit 0 | SB | START 已发送 |
| bit 1 | ADDR | 地址已发送且收到 ACK |
| bit 2 | BTF | 字节传输完成 |
| bit 6 | RXNE | 接收数据寄存器非空 |
| bit 7 | TXE | 发送数据寄存器空 |
| bit 10 | AF | 应答失败（NACK） |

### 4.6 `I2C1->SR2`

包含 BUSY 标志（bit 1），并用于配合清除 ADDR 标志。

### 4.7 清除 ADDR 标志的特殊操作

```c
temp = I2C1->SR1;  // 读 SR1
temp = I2C1->SR2;  // 读 SR2 → 硬件自动清 ADDR
```

这是 STM32F103 I2C 硬件的规定操作，缺一不可。

## 5. 寄存器版写 EEPROM 流程

```
START → 地址+写位(0xA0) → 等待 ADDR → 清除 ADDR
  → 等待 TXE → 发内存地址(0x00) → 等待 BTF
  → 发数据字节 → 等待 BTF → STOP
```

## 6. 寄存器版读 EEPROM 流程（随机读）

```
[第 1 段：写内存地址]
START → 地址+写位(0xA0) → 清 ADDR → 发内存地址(0x00) → 等 BTF

[第 2 段：重复起始 + 读取]
RESTART → 地址+读位(0xA1) → 清 ADDR（先关 ACK + 发 STOP）
  → 等 RXNE → 读 DR → 恢复 ACK
```

单字节接收的关键：收到地址应答后，先关 ACK 再清 ADDR，然后提前发 STOP。这样从机发完一个字节后主不应答，总线释放。

## 7. HAL 版关键 API

### 7.1 `HAL_I2C_Mem_Write()`

```c
HAL_I2C_Mem_Write(&hi2c1, addr, mem_addr, mem_size, data, len, timeout);
```

封装了整个 I2C 写操作状态机，适合 EEPROM 等带内部地址的 I2C 设备。

### 7.2 `HAL_I2C_Mem_Read()`

```c
HAL_I2C_Mem_Read(&hi2c1, addr, mem_addr, mem_size, data, len, timeout);
```

封装了"写内存地址 → 重复起始 → 读取"的完整流程。

## 8. 本课最容易出错的地方

| 问题 | 后果 | 解决 |
|------|------|------|
| 没有上拉电阻 | 通信失败 | 加 4.7kΩ 上拉到 3.3V |
| A0/A1/A2 不接地 | 地址不对 | 确认 7 位地址 = 0x50 |
| WP 没接地 | 写不进去 | WP 接 GND |
| PB6/PB7 配成推挽 | 可能短路 | 配成复用开漏（AF_OD） |
| 写完没等写周期 | 读回旧值 | 等 10ms 或轮询 ACK |
| FREQ 配错 | 时序全错 | 确认 FREQ=PCLK1(MHz) |

## 9. 本课你真正应该掌握什么

1. I2C 是两线地址总线，靠地址区分设备
2. I2C 依赖**开漏 + 上拉**的电气结构
3. `0x50` 和 `0xA0/0xA1` 的关系是"7 位地址 + R/W 位"
4. EEPROM 随机读需要先写内部地址再用**重复起始**切到读方向
5. 清除 ADDR 标志需先读 SR1 再读 SR2
6. HAL 的 `Mem_Write/Read` 封装了整套 I2C 状态机
