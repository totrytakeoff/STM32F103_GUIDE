## 5. 先建立一个最基本的脑图

本课整条链路如下：

1. 先把系统时钟配置到 72MHz
2. 给 `TIM2` 打开外设时钟
3. 设定 `TIM2` 的输入计数频率
4. 让它从 0 开始不断计数
5. 当计数到 `ARR` 指定值时，产生一次更新事件
6. 如果更新中断开启，就进入 `TIM2_IRQHandler()`
7. 在中断里翻转 LED

这条链路里最重要的两个参数就是：

- `PSC`
- `ARR`

因为：

- `PSC` 决定计数器“数得多快”
- `ARR` 决定“数到多少算一次周期结束”

## 6. 先认识本课里出现的核心名词

### 6.1 TIM2 是什么

`TIM2` 是 STM32F103 上的一个通用定时器。

你可以先把它理解成：

- 一个可以不断计数的硬件计数器模块

它和 `SysTick` 的区别在于：

- `SysTick` 是内核自带的系统定时器
- `TIM2` 是片上普通外设定时器

后面你学 PWM、输入捕获、输出比较时，很多内容都建立在这类定时器之上。

### 6.2 `TIM2->PSC` 是什么

`PSC` 是：

- Prescaler，预分频器

它的作用是：

- 先把定时器输入时钟分频

例如：

- 如果定时器时钟是 72MHz
- `PSC = 7199`

那么计数频率就是：

- `72MHz / (7199 + 1) = 10kHz`

也就是说：

- 定时器以后就按 10kHz 的速度去加 1

### 6.3 `TIM2->ARR` 是什么

`ARR` 是：

- Auto-Reload Register，自动重装寄存器

它的作用可以先理解成：

- 计数上限

当 `CNT` 一直往上数，数到 `ARR` 之后，就会产生一次更新事件，然后重新开始下一轮。

### 6.4 `TIM2->CNT` 是什么

`CNT` 是当前计数器的值。

也就是说：

- 它记录“现在已经数到哪里了”

### 6.5 什么是更新事件

当定时器计数到达设定边界时，会产生一次：

- 更新事件

你可以把它理解成：

- “本轮计时结束了”

如果开启了更新中断，那么这次事件就会进一步触发中断服务函数。

### 6.6 `TIM2->DIER` 是什么

`DIER` 是：

- DMA/Interrupt Enable Register

本课里你只需要先关注其中一个位：

- `UIE`

也就是：

- Update Interrupt Enable，更新中断使能

它决定：

- 更新事件发生时，要不要触发中断

### 6.7 `TIM2->SR` 是什么

`SR` 是：

- Status Register，状态寄存器

本课里你只需要先关注：

- `UIF`

也就是：

- Update Interrupt Flag，更新中断标志

它表示：

- 已经发生过一次更新事件

### 6.8 `TIM2->CR1` 是什么

`CR1` 是：

- Control Register 1，控制寄存器 1

本课最关键的是：

- `CEN`

也就是：

- Counter Enable，计数器使能

只有把它打开，定时器才真正开始跑。

### 6.9 为什么 TIM2 明明在 APB1，却常常跑到 72MHz

这是本课最关键的知识点之一。

你上一课已经知道当前时钟配置是：

- `HCLK = 72MHz`
- `PCLK1 = 36MHz`

而 `TIM2` 挂在：

- `APB1`

直觉上你可能会以为：

- `TIM2` 时钟就是 36MHz

但在 STM32F1 里，存在一个定时器时钟规则：

- 如果 APB 预分频系数等于 1，那么定时器时钟 = PCLK
- 如果 APB 预分频系数不等于 1，那么定时器时钟 = 2 x PCLK

当前我们的 `APB1` 分频是：

- `/2`

所以：

- `PCLK1 = 36MHz`
- `TIM2CLK = 2 x 36MHz = 72MHz`

这条规则必须吃透，因为后面所有定时器频率计算都要用它。

### 6.10 NVIC 在本课里扮演什么角色

定时器产生更新事件，只是“外设侧”准备好了中断请求。

要想真正让 CPU 跳进中断函数，还必须让：

- NVIC 接受并分发这个中断

所以定时器中断一定有两层：

1. 外设自己允许更新中断
2. NVIC 允许 `TIM2_IRQn`

少任何一层都不行。

### 6.11 `HAL_TIM_Base_Init()` 是什么

这是 HAL 里用于初始化定时器基础计数功能的函数。

它主要负责：

- 根据 `TIM_HandleTypeDef` 里的参数
- 配置预分频
- 配置计数模式
- 配置自动重装值

也就是说，它是在封装：

- `PSC`
- `ARR`
- `CR1` 中部分计数基础配置

### 6.12 `HAL_TIM_Base_Start_IT()` 是什么

它的作用是：

- 开启定时器基础计数
- 同时开启更新中断

所以它相当于同时完成了：

- 定时器启动
- 中断打开

### 6.13 `HAL_TIM_PeriodElapsedCallback()` 是什么

这是 HAL 提供的定时器周期到达回调函数。

当 HAL 的定时器中断处理函数发现：

- 某个定时器发生了更新事件

就会回调到这里。

所以你可以先把它理解成：

- “HAL 风格的定时器到点处理函数”

## 7. 本课 Demo 要实现什么

本课最终 Demo 是：

- 使用 `TIM2` 每 1 秒触发一次中断
- 每次中断翻转一次板载 LED

### 7.1 本课选用的频率设计

本课我们会这样设计：

1. `TIM2` 输入时钟 = `72MHz`
2. 通过 `PSC = 7200 - 1`，把计数频率降到 `10kHz`
3. 再设置 `ARR = 10000 - 1`
4. 这样每数满 `10000` 次，就过了 1 秒

计算过程如下：

- `72MHz / 7200 = 10kHz`
- `10kHz / 10000 = 1Hz`

也就是：

- 每 1 秒产生一次更新事件

## 8. 寄存器版代码逐步讲解

对应代码文件：

- [reg/src/main.c](/home/myself/workspace/mcu/stm32/STM32F103C8T6/05_timer_base/reg/src/main.c)

### 8.1 寄存器版主线

1. 配系统时钟到 72MHz
2. 配置 LED
3. 打开 TIM2 时钟
4. 配 `PSC` 和 `ARR`
5. 清更新标志
6. 开更新中断
7. 开 NVIC
8. 启动定时器
9. 在 `TIM2_IRQHandler()` 中翻转 LED

### 8.2 为什么要先清 `UIF`

因为在某些配置过程中，更新标志可能已经被置位。

如果你不先清掉它，就可能出现：

- 刚开中断就立刻进一次中断

不利于理解“真正的周期性触发”。

### 8.3 为什么中断里要先判断 `UIF`

因为一个中断服务函数理论上应该先确认：

- 到底是不是这类事件触发的

虽然本课只处理更新事件，但这个习惯很重要。

## 9. HAL版代码逐步讲解

对应代码文件：

- [hal/src/main.c](/home/myself/workspace/mcu/stm32/STM32F103C8T6/05_timer_base/hal/src/main.c)

### 9.1 HAL 版里为什么会出现 `TIM_HandleTypeDef`

这是 HAL 用来描述和管理一个定时器实例的句柄结构体。

你可以先把它理解成：

- HAL 眼里的“这个定时器对象”

里面会记录：

- 是哪个定时器
- 采用什么基础配置

### 9.2 `htim2.Init.Prescaler` 是什么

这是 HAL 对应：

- `PSC`

的封装字段。

### 9.3 `htim2.Init.Period` 是什么

这是 HAL 对应：

- `ARR`

的封装字段。

### 9.4 `HAL_TIM_IRQHandler()` 为什么要在 `TIM2_IRQHandler()` 里调用

因为 HAL 并不会自己替你生成中断函数体。

真正的中断入口函数还是：

- `TIM2_IRQHandler()`

而 HAL 提供的是：

- 一个通用的定时器中断处理函数

所以标准写法是：

1. 中断来了先进入 `TIM2_IRQHandler()`
2. 再把控制权交给 `HAL_TIM_IRQHandler(&htim2)`
3. HAL 在内部分析标志位并调用回调函数

### 9.5 为什么最终在 `HAL_TIM_PeriodElapsedCallback()` 里处理 LED

因为本课用的是：

- 更新中断

对于 HAL 来说，这类事件最终最自然的落点就是：

- 周期到达回调函数

## 10. 寄存器版和 HAL版对比

### 10.1 寄存器版重点看什么

- `TIM2` 的时钟来源
- `PSC` 和 `ARR` 的频率计算
- `DIER / SR / CR1` 的职责
- 外设中断和 NVIC 的双层打开逻辑

### 10.2 HAL版重点看什么

- `TIM_HandleTypeDef` 如何表达一个定时器
- `Prescaler`、`Period` 和底层寄存器的对应关系
- HAL 如何通过中断入口函数和回调函数组织逻辑

### 10.3 两者如何一一对应

- 寄存器版写 `PSC` <-> HAL 的 `Init.Prescaler`
- 寄存器版写 `ARR` <-> HAL 的 `Init.Period`
- 寄存器版写 `DIER/UIE` 和 `CR1/CEN` <-> HAL 的 `HAL_TIM_Base_Start_IT()`
- 寄存器版写 `TIM2_IRQHandler()` <-> HAL 的 `TIM2_IRQHandler() + HAL_TIM_IRQHandler() + 回调`

## 12. 常见问题排查

### 12.1 LED 完全不闪

优先检查：

1. `TIM2` 时钟是否打开
2. `CEN` 是否置位
3. `UIE` 是否置位
4. NVIC 是否使能了 `TIM2_IRQn`
5. `TIM2_IRQHandler()` 是否正确实现

### 12.2 闪得太快或太慢

优先检查：

1. 定时器输入时钟是否真的按 `72MHz` 计算
2. 是否忘了 APB1 定时器倍频规则
3. `PSC` 和 `ARR` 是否算错

### 12.3 HAL 版里回调不进

优先检查：

1. 是否调用了 `HAL_TIM_Base_Start_IT()`
2. `TIM2_IRQHandler()` 里是否调用了 `HAL_TIM_IRQHandler(&htim2)`
3. `htim2.Instance` 是否真的是 `TIM2`

## 13. 本课要点总结

本课最核心的结论如下：

1. 通用定时器和 `SysTick` 是两类不同的定时器
2. `PSC` 决定计数速度，`ARR` 决定周期长度
3. 定时器到达 `ARR` 后会产生更新事件
4. 更新事件配合 `UIE` 和 NVIC，就能形成周期中断
5. 当前配置下 `TIM2` 输入时钟是 `72MHz`，不是 `36MHz`
6. HAL 对定时器的封装，本质仍然是围绕 `PSC / ARR / 中断` 这些底层概念
