# 第 7 课：PWM 进阶

## 1. 本课到底在学什么

上一课你已经能做出一个基础 PWM：

- 定时器输出 PWM 波形
- 动态修改 `CCR`
- LED 亮度跟着变化

但如果你真的把上一课的代码跑起来，你会发现一个很现实的问题：

- 亮度变化虽然有了
- 但不一定“好看”
- 有时候变化太生硬
- 有时候接近最亮或最暗时不够细腻

所以本课真正要学的，不再是：

- PWM 能不能输出

而是：

- 占空比应该怎么动态变化
- 什么叫“线性占空比变化”
- 为什么线性修改占空比，视觉上不一定线性
- 什么叫呼吸灯
- 如何用更合适的步进策略，让亮度变化更自然

本课最终会做一个更像样的：

- 呼吸灯

## 5. 先建立一个最基本的脑图

本课整条链路如下：

1. 继续沿用上一课的 PWM 输出链路
2. 保持 PWM 频率稳定不变

这节课最重要的理解点是：

- PWM 调光分成两个层次

- 硬件层：PWM 波形怎么产生

第二层：

- 策略层：占空比怎么随时间变化

上一课重点是第一层。

这一课重点是第二层。

## 6. 先认识本课里出现的核心名词

### 6.1 什么叫“呼吸灯”

呼吸灯本质上是：

- 亮度慢慢变亮
- 再慢慢变暗
- 周期性重复

它和“简单亮度跳变”最大的区别在于：

- 变化过程应该是连续而柔和的

### 6.2 什么叫“线性占空比变化”

比如：

- 每次把 `CCR1` 增加 10
- 再每次减少 10

这就是一种线性的数值变化。

因为：

- 每一步变化量相同

### 6.3 为什么线性占空比变化，视觉上不一定线性

因为人眼对亮度的感知不是完全线性的。

这意味着：

- 从 5% 提到 10%
- 和从 85% 提到 90%

虽然数值变化一样大，但主观感觉不一定一样。

所以真正“看起来顺滑”的呼吸灯，很多时候不会只用简单线性步进。

### 6.4 本课用什么方式做更自然的变化

为了兼顾：

- 教学清晰
- 代码可读
- 现象更自然

本课采用一种比较适合入门的折中策略：

- 在暗部使用更小步进
- 在中高亮区域使用更大步进

这样可以让：

- 低亮度变化不至于太突兀
- 整体节奏也不会太慢

### 6.5 `CCR1` 在本课中的角色

这一课你要把 `CCR1` 彻底当成：

- 占空比控制杆

因为：

- `ARR` 已经固定了周期长度
- `CCR1` 决定当前高电平宽度

所以这节课里所有“呼吸效果”的核心，本质上都是：

- 如何随时间修改 `CCR1`

### 6.6 为什么本课不先引入 Gamma 校正表

更高级的亮度控制，常常会使用：

- Gamma 校正表
- 查表法
- 非线性映射

这些都很有价值。

但如果现在一下子把它们全塞进来，主线会变乱。

所以本课先停在：

- 可直接理解的分段步进策略

后面如果需要，我们可以单独做“亮度映射和感知优化”的专题。

## 7. 本课 Demo 要实现什么

本课会继续使用：

- `TIM2_CH1 -> PA0`

输出 PWM。

### 7.1 PWM 基础参数保持不变

为了让本课只聚焦“亮度变化策略”，PWM 基本参数继续保持和上一课一致：

- 定时器输入时钟按 72MHz 计算
- `PSC = 72 - 1`
- `ARR = 1000 - 1`
- PWM 频率 = 1kHz

### 7.2 本课变化的只有一件事

本课不会改 PWM 频率，而只改：

- `CCR1` 的更新策略

这样你就能更清楚地知道：

- 亮度变自然，是因为“策略更好了”
- 不是因为底层 PWM 配置换了

## 8. 寄存器版代码逐步讲解

对应代码文件：

- [reg/src/main.c](/home/myself/workspace/mcu/stm32/STM32F103C8T6/07_pwm_advanced/reg/src/main.c)

### 8.1 寄存器版主线

1. 复用上一课的 PWM 基础初始化
2. 继续让 `PA0` 输出 1kHz PWM
3. 主循环中不再简单固定步进
4. 而是根据当前亮度区间，选择不同的步进值

### 8.2 为什么要分区间步进

例如本课会把占空比区间大致分成：

- 暗区
- 中间区
- 亮区

这样做的目的就是：

- 暗区变化更细
- 中亮区变化稍快
- 整体效果更柔和

### 8.3 为什么还要有方向变量

因为呼吸灯必须完成：

- 变亮
- 再变暗

所以主循环里需要一个方向变量，通常可以理解成：

- `+1`：往更亮方向走
- `-1`：往更暗方向走

### 9.1 HAL 版本课重点看什么

本课 HAL 版的重点已经不再是：

- PWM 初始化怎么写

因为这部分上一课已经讲过。

本课重点变成：

- 如何在应用层优雅地动态更新占空比

### 9.2 为什么 `__HAL_TIM_SET_COMPARE()` 非常适合做呼吸灯

因为它本质上就是：

- 改某个通道当前比较值

也就是：

- 改 `CCR`

这正好对应本课最核心的控制量。

所以 HAL 版做动态亮度控制时，这个宏非常直接、非常适合。

## 10. 寄存器版和 HAL版对比

### 10.1 寄存器版重点看什么

- 动态改 `CCR1` 就是在动态改占空比
- 不同步进策略会带来不同视觉效果
- “PWM 初始化”和“PWM 亮度策略”是两个层次

### 10.2 HAL版重点看什么

- 如何把动态占空比更新写成更工程化的代码
- 用 HAL 接口快速调试和修改效果

### 10.3 两者如何一一对应

- 寄存器版写 `TIM2->CCR1 = x` <-> HAL 版 `__HAL_TIM_SET_COMPARE(..., x)`
- 寄存器版的占空比更新策略 <-> HAL 版相同的策略，只是换了接口层

## 11. 运行现象

如果程序正常，烧录后你会看到：

- 接在 `PA0` 上的外部 LED 亮度缓慢上升
- 再缓慢下降
- 周期性重复

而且相比上一课，呼吸效果应该更自然，不那么“生硬跳变”。

## 12. 常见问题排查

### 12.1 LED 有亮度变化，但还是很突兀

优先检查：

1. 步进值是否太大
2. 每次更新间隔是否太短或太长
3. 是否区间划分不合理

### 12.2 LED 一直很暗或一直很亮

优先检查：

1. `CCR1` 是否被限制在合理范围
2. 是否不小心让 `CCR1` 长时间接近 0 或接近 `ARR`
3. LED 接线是否正确

### 12.3 看不出呼吸感

优先检查：

1. PWM 频率是否稳定
2. 占空比变化幅度是否覆盖足够大范围
3. 更新节奏是否过快

## 13. 本课要点总结

本课最核心的结论如下：

1. 呼吸灯的关键不只是“有 PWM”，还包括“占空比怎么变化”
2. `CCR1` 是本课最核心的动态控制量
3. 线性数值变化不一定带来线性的视觉亮度变化
4. 通过更合理的步进策略，可以让亮度变化更自然
5. PWM 初始化属于底层，亮度变化策略属于应用层
