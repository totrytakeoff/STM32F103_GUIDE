# 第 29 课：I2C 读写 MPU6050

## 1. 本课到底在学什么

本课表面现象是：STM32 通过 I2C1 唤醒 MPU6050，并周期读取 `WHO_AM_I` 身份寄存器；如果读到 `0x68`，PC13 LED 快速闪烁，否则慢速闪烁。

真正要学的是：I2C 传感器通常不是“直接读数据”，而是先访问器件内部寄存器。你要先发送器件地址，再发送寄存器地址，然后读写寄存器内容。

本课链路是：

```text
PB6/PB7 I2C1
  -> MPU6050 地址 0x68
  -> 写 PWR_MGMT_1(0x6B) = 0x00 唤醒
  -> 读 WHO_AM_I(0x75)
  -> 返回值应为 0x68
  -> 根据 ID 控制 PC13 闪烁快慢
```

这节课不是完整姿态解算课，也没有读取加速度和陀螺仪数据。它先教你验证一个 I2C 传感器是否在线、地址是否正确、寄存器读写是否通。

## 2. 本课学习目标

学完本课，你应该能回答：

1. MPU6050 的 7 位地址 `0x68` 为什么在代码里写成 `0xD0/0xD1`？
2. `PWR_MGMT_1` 为什么上电后要写 `0x00`？
3. `WHO_AM_I` 为什么适合作为 I2C 通信自检寄存器？
4. 读 MPU6050 寄存器为什么需要“先写寄存器地址，再重复起始读数据”？
5. `HAL_I2C_Mem_Read()` 的 `I2C_MEMADD_SIZE_8BIT` 为什么不能乱选？
6. ID 读不到 `0x68` 时应该优先排查哪几层？
7. 本课 LED 快慢闪分别代表什么？

## 4. 实验硬件

- 开发板：STM32F103C8T6 BluePill
- 下载器：ST-Link
- 传感器：MPU6050 模块
- I2C 引脚：PB6 = SCL，PB7 = SDA
- LED：PC13

接线：

```text
MPU6050 VCC -> 3.3V
MPU6050 GND -> GND
MPU6050 SCL -> PB6
MPU6050 SDA -> PB7
```

多数 MPU6050 模块 AD0 接低电平时 7 位地址为 `0x68`。若 AD0 接高电平，地址会变为 `0x69`，代码中的地址也要相应修改。

## 5. 先建立一个最基本的脑图

```text
I2C1 初始化
  -> PB6/PB7 复用开漏
  -> 100kHz 标准模式

唤醒 MPU6050
  -> START
  -> 地址+写 0xD0
  -> 寄存器地址 0x6B
  -> 数据 0x00
  -> STOP

读取 WHO_AM_I
  -> START
  -> 地址+写 0xD0
  -> 寄存器地址 0x75
  -> RESTART
  -> 地址+读 0xD1
  -> 读 1 字节
  -> STOP

判断结果
  -> id == 0x68：快速闪烁
  -> id != 0x68：慢速闪烁
```

这节课的关键是把“传感器内部寄存器地址”和“I2C 器件地址”分清楚。`0x68` 是器件地址/身份值，`0x75` 是 WHO_AM_I 寄存器地址，`0x6B` 是电源管理寄存器地址。

## 6. 先认识本课里出现的核心名词

### 6.1 `MPU6050` 是什么

MPU6050 是一颗 6 轴惯性传感器，内部包含三轴加速度计和三轴陀螺仪。

它属于外部传感器层。STM32 通过 I2C 访问它内部寄存器，先验证身份，再配置工作状态，后续才能读取测量数据。

如果模块没供电、地址不对或没上拉，STM32 读不到正确 ID。

### 6.2 `7 位地址 0x68` 是什么

MPU6050 常见 7 位 I2C 地址是 `0x68`。I2C 线上发送地址字节时要左移一位并加入 R/W 位：

```text
写地址字节：0x68 << 1 | 0 = 0xD0
读地址字节：0x68 << 1 | 1 = 0xD1
```

寄存器版定义 `MPU_ADDR_W 0xD0` 和 `MPU_ADDR_R 0xD1`。HAL 版传入 `MPU_ADDR 0xD0`，HAL 根据读写方向处理最低位。

### 6.3 `PWR_MGMT_1` 是什么

`PWR_MGMT_1` 是 MPU6050 的电源管理寄存器，地址是 `0x6B`。

很多 MPU6050 上电后处于睡眠状态。代码写：

```c
mpu_write(0x6B, 0x00);
```

含义是清掉睡眠相关设置，让传感器进入工作状态。

如果不唤醒，读身份寄存器仍可能成功，但后续加速度/陀螺仪数据不会按你预期更新。

### 6.4 `WHO_AM_I` 是什么

`WHO_AM_I` 是 MPU6050 的身份寄存器，地址是 `0x75`。

正常读取应得到 `0x68`。它适合做通信自检，因为不需要复杂配置，只要器件在线、地址正确、寄存器读流程正确，就能读到稳定值。

如果读到 `0xFF`、`0x00` 或随机值，通常说明 I2C 物理层或地址/读流程有问题。

### 6.5 `寄存器地址` 是什么

MPU6050 内部有很多 8 位寄存器地址，如 `0x6B`、`0x75`。

它和 I2C 器件地址不是一回事。I2C 地址用来选中“哪个器件”，寄存器地址用来选中“这个器件内部哪个功能单元”。

把这两层混淆，是传感器 I2C 初学时最常见的问题。

可以把 MPU6050 看成一个有门牌号的房子，I2C 器件地址 `0x68` 是门牌号，内部寄存器地址 `0x75` 是房间号。总线上先用门牌号找到这颗芯片，再把房间号发给它，芯片才知道你要访问哪个内部寄存器。

本课的 `0x68` 还容易和 WHO_AM_I 返回值混在一起。`0x68` 既是常见 7 位器件地址，也是 WHO_AM_I 正常返回值；但 `0x75` 才是 WHO_AM_I 的寄存器地址。读代码时要把“地址字节、寄存器地址、寄存器内容”三列分开写。

### 6.6 `重复起始` 是什么

读取 MPU6050 某个寄存器时，主机先用写方向发送寄存器地址，再用重复起始切换到读方向。

流程是：

```text
START -> 0xD0 -> reg
RESTART -> 0xD1 -> data
```

它属于 I2C 协议时序层。没有先写寄存器地址，MPU6050 不知道你想读哪个寄存器。

重复起始和 STOP 后再 START 不完全一样。许多寄存器型 I2C 器件希望主机在写完内部寄存器地址后，不释放总线，直接用 repeated START 切到读方向，这样器件内部地址指针保持在刚设置的位置。

寄存器版 `mpu_read()` 里先 `start_addr(MPU_ADDR_W)`、写 `reg`，然后再次 `start_addr(MPU_ADDR_R)`。第二次 START 就是重复起始。HAL 版 `HAL_I2C_Mem_Read()` 把这个序列封装起来，但底层仍然是同样的协议动作。

### 6.7 `ACK` 是什么

ACK 是从机收到地址或数据后的应答。MPU6050 地址匹配时会 ACK 地址字节。

若地址阶段没有 ACK，寄存器版会卡在等待 `ADDR`，HAL 版会返回错误或超时。常见原因是地址不对、模块没接、SDA/SCL 没上拉、AD0 电平与代码不匹配。

### 6.8 `I2C1` 是什么

I2C1 是 STM32 内部 I2C 外设，使用 PB6/PB7 作为默认 SCL/SDA。

它属于芯片外设层，负责产生 START/STOP、发送地址、等待 ACK、收发数据。它不理解 `PWR_MGMT_1` 或 `WHO_AM_I` 的含义，这些是 MPU6050 协议层内容。

### 6.9 `I2C_MEMADD_SIZE_8BIT` 是什么

HAL 版读写 MPU6050 时传入 `1`，等价于 8 位寄存器地址长度。

MPU6050 的寄存器地址是 8 位，所以 HAL 只能发送 1 个地址字节。如果误用 16 位地址长度，HAL 会多发一个地址字节，MPU6050 解释就会错。

### 6.10 `HAL_I2C_Mem_Write()` 是什么

HAL 写寄存器 API。当前调用：

```c
HAL_I2C_Mem_Write(&hi2c1, MPU_ADDR, 0x6B, 1, &zero, 1, 100);
```

表示向器件 `0xD0` 的内部寄存器 `0x6B` 写入 1 字节 `0x00`。

### 6.11 `HAL_I2C_Mem_Read()` 是什么

HAL 读寄存器 API。当前调用：

```c
HAL_I2C_Mem_Read(&hi2c1, MPU_ADDR, 0x75, 1, &id, 1, 100);
```

它封装“写寄存器地址 + 重复起始 + 读数据”的完整流程。

### 6.12 `PC13 快慢闪` 是什么

PC13 是本课的现象层输出。代码根据 `id==0x68` 选择延时：

- ID 正确：延时短，LED 快闪
- ID 错误：延时长，LED 慢闪

它不是传感器数据，只是通信自检结果的可见指示。

## 7. 寄存器版代码逐步讲解

### 7.1 时钟和 LED

系统时钟配置为 72MHz，I2C1 所在 APB1 为 36MHz。PC13 初始化为推挽输出，用来显示 ID 判断结果。

### 7.2 `i2c1_init()` 配置 I2C1

代码打开 GPIOB 和 I2C1 时钟，把 PB6/PB7 配为复用开漏，然后设置：

```c
I2C1->CR2 = 36;
I2C1->CCR = 180;
I2C1->TRISE = 37;
I2C1->CR1 = I2C_CR1_PE;
```

这对应 36MHz PCLK1 下的 100kHz I2C 标准模式。

### 7.3 `start_addr()`

函数设置 `START`，等待 `SB`，写地址字节，再等待 `ADDR`，最后读 `SR1/SR2` 清地址标志。

这一步用于选中 MPU6050。如果地址错，程序会卡在 `ADDR` 等待处。

### 7.4 为什么要读 `SR1/SR2`

STM32F103 的 I2C `ADDR` 标志不能随便写 0 清除。地址阶段完成后，软件必须先读 `SR1`，再读 `SR2`。

代码里的：

```c
(void)I2C1->SR1;
(void)I2C1->SR2;
```

就是在执行这个硬件规定动作。若不清 `ADDR`，I2C 外设会停在地址阶段，后续写寄存器地址或读取数据都可能无法继续。

### 7.5 `wr()` 写一个字节

```c
while(!(I2C1->SR1&I2C_SR1_TXE)){}
I2C1->DR=b;
```

等待发送寄存器空，再写一个字节。这个字节可能是 MPU6050 寄存器地址，也可能是要写入寄存器的数据。

### 7.6 `mpu_write()`

```c
start_addr(MPU_ADDR_W);
wr(reg);
wr(val);
while(!(I2C1->SR1&I2C_SR1_BTF)){}
I2C1->CR1|=I2C_CR1_STOP;
```

它完成一次寄存器写操作：选中器件、发送寄存器地址、发送数据、等待传输完成、STOP。

本课用它写 `0x6B=0x00` 唤醒 MPU6050。

### 7.7 `mpu_read()` 第一段

```c
start_addr(MPU_ADDR_W);
wr(reg);
while(!(I2C1->SR1&I2C_SR1_BTF)){}
```

先用写方向告诉 MPU6050 要读哪个寄存器。这里传入 `0x75`，表示 WHO_AM_I。

### 7.8 `mpu_read()` 第二段

```c
I2C1->CR1 &= ~I2C_CR1_ACK;
start_addr(MPU_ADDR_R);
I2C1->CR1 |= I2C_CR1_STOP;
while(!(I2C1->SR1&I2C_SR1_RXNE)){}
v=(uint8_t)I2C1->DR;
I2C1->CR1 |= I2C_CR1_ACK;
```

关闭 ACK 是因为只读 1 个字节。随后重复起始切换到读方向，产生 STOP，等待 `RXNE`，读取数据，再恢复 ACK。

这段是单字节 I2C 读里最容易出错的地方。

F103 I2C 单字节主接收有严格顺序：在地址阶段后接收一个字节时，主机要提前准备 NACK 和 STOP，让从机知道“这就是最后一个字节”。本课代码在切换读方向前清 `ACK`，地址确认后很快置 `STOP`，再等 `RXNE` 读取 `DR`。

如果 ACK 关闭太晚，从机可能以为主机还要继续读；如果 STOP 太早或太晚，总线可能保持 BUSY，下一次通信卡住。看到第一次读成功、后续读卡住时，这段 ACK/STOP 时序就是重点检查对象。

### 7.9 单字节读为什么要关 ACK

I2C 读多个字节时，主机通常对前面的字节 ACK，表示“继续发”。但本课只读一个字节，主机要在这一个字节后结束，所以需要提前关闭 ACK，让从机收到 NACK 后停止继续发送。

如果 ACK/STOP 时序写错，常见现象是读操作卡住、总线 BUSY 不释放，或者下一次读寄存器时状态异常。

### 7.10 `main()` 唤醒传感器

```c
mpu_write(0x6B,0x00);
```

这一步写 `PWR_MGMT_1`，让 MPU6050 离开睡眠。后续若扩展读取加速度和陀螺仪，这一步就很重要。

### 7.11 `main()` 读取 ID 并控制 LED

```c
uint8_t id=mpu_read(0x75);
pc13_toggle();
delay_cycles((id==0x68U)?720000U:3600000U);
```

读到 `0x68` 说明通信链路基本正确，LED 快闪；否则慢闪。

## 8. HAL 版代码逐步讲解

### 8.1 HAL 初始化

HAL 版调用 `HAL_Init()`、配置 72MHz 时钟、初始化 PC13 和 I2C1。PB6/PB7 配成 `GPIO_MODE_AF_OD`，与寄存器版复用开漏一致。

### 8.2 `hi2c1` 字段

`hi2c1.Instance = I2C1` 绑定 I2C1，`ClockSpeed=100000` 配成 100kHz，`AddressingMode=I2C_ADDRESSINGMODE_7BIT` 表示使用 7 位地址模式。

### 8.3 `HAL_I2C_Init()`

该函数根据 `hi2c1.Init` 配置 I2C1 的时序和控制寄存器，对应寄存器版 `CR2/CCR/TRISE/CR1`。

### 8.4 唤醒 MPU6050

```c
uint8_t zero=0;
HAL_I2C_Mem_Write(&hi2c1,MPU_ADDR,0x6B,1,&zero,1,100);
```

这表示向 `PWR_MGMT_1` 写 `0x00`。参数中的 `1` 是内存地址长度 1 字节，也就是 8 位寄存器地址。

### 8.5 读取 WHO_AM_I

```c
HAL_I2C_Mem_Read(&hi2c1,MPU_ADDR,0x75,1,&id,1,100);
```

HAL 内部先发送寄存器地址 `0x75`，再重复起始读取 1 字节。读到的值放入 `id`。

### 8.6 HAL 版地址格式

`MPU_ADDR` 定义为 `0xD0`，即 7 位地址 `0x68` 左移一位。HAL F1 I2C API 需要传入左移后的地址。

如果 AD0 接高，7 位地址为 `0x69`，这里应改成 `0xD2`。

### 8.7 HAL 返回值

当前 HAL 版没有检查 `HAL_I2C_Mem_Write/Read` 返回值。学习时建议你加上判断，失败时点亮 LED 或进入错误处理，这样排错更清楚。

`HAL_I2C_Mem_Read()` 如果失败，`id` 可能仍保持上一次值或初始值。当前代码只看 `id == 0x68`，所以一次失败可能被误判成慢闪，而你不知道失败发生在地址阶段、寄存器地址阶段还是读数据阶段。

工程写法通常是：

```c
if (HAL_I2C_Mem_Read(...) != HAL_OK) {
    error_handler();
}
```

然后在错误处理里查看 `hi2c1.ErrorCode`。这样可以把“通信 API 失败”和“通信成功但返回值不是 0x68”区分开。

### 8.9 HAL 的寄存器地址长度参数

源码里 `HAL_I2C_Mem_Write(&hi2c1, MPU_ADDR, 0x6B, 1, &zero, 1, 100)` 第 4 个参数写 `1`，在当前 HAL 头文件里等价于 8 位内部地址长度。

更可读的写法通常是 `I2C_MEMADD_SIZE_8BIT`。这个参数控制 HAL 在器件地址之后发送几个字节作为内部寄存器地址。MPU6050 的寄存器地址是 8 位，发 1 个字节正好；如果误发 2 个字节，MPU6050 会把多出来的字节当成错误数据或错误寄存器流程。

### 8.10 HAL_Delay 与现象判断

HAL 版用 `HAL_Delay((id==0x68U)?100:500)` 把通信结果变成 LED 快慢闪。这个现象依赖 SysTick 正常工作，也依赖 `HAL_Init()` 已经配置 tick。

如果 LED 完全不闪，不要急着判断 MPU6050 坏了，先确认程序是否卡在 I2C API、SysTick 是否运行、PC13 是否初始化。只有 LED 能按快慢变化，才说明主循环持续执行到了判断逻辑。

### 8.8 LED 快慢闪

HAL 版使用：

```c
HAL_Delay((id==0x68U)?100:500);
```

ID 正确时约 100ms 翻转一次，错误时约 500ms 翻转一次。

## 9. 两个版本真正应该怎么学

寄存器版重点看 I2C 单字节寄存器读写的底层顺序；HAL 版重点看 `HAL_I2C_Mem_*` 如何把“器件地址、寄存器地址、数据缓冲区”组合起来。

本课要建立传感器访问模型：

```text
器件地址选中模块
寄存器地址选中模块内部功能
数据字节读写这个功能的状态或配置
```

## 10. 检验问题清单

### 10.1 `0x68` 和 `0xD0` 是什么关系？

**答**：`0x68` 是 MPU6050 的 7 位地址，`0xD0` 是它左移一位并加写位后的地址字节。

### 10.2 `WHO_AM_I` 为什么读到 `0x68`？

**答**：MPU6050 的身份寄存器设计为返回器件身份值。读到 `0x68` 说明地址、接线和寄存器读流程基本正确。

### 10.3 为什么要写 `PWR_MGMT_1=0x00`？

**答**：MPU6050 上电后常处于睡眠状态，写 `0x00` 用于唤醒传感器，为后续正常采样做准备。

### 10.4 读寄存器为什么先用写方向？

**答**：因为要先告诉 MPU6050 要读哪个内部寄存器。发送寄存器地址属于写方向，随后再重复起始切到读方向取数据。

### 10.5 `I2C_MEMADD_SIZE_8BIT` 控制什么？

**答**：它控制 HAL 发送几个字节作为内部寄存器地址。MPU6050 寄存器地址是 8 位，所以应选择 8 位。

### 10.6 LED 慢闪说明什么？

**答**：说明当前读到的 ID 不是 `0x68`。可能是模块未响应、地址错、读流程错，也可能是读到了错误值。

### 10.7 如果 AD0 接高要改哪里？

**答**：7 位地址从 `0x68` 变成 `0x69`。寄存器版写地址应改为 `0xD2`、读地址改为 `0xD3`；HAL 版地址改为 `0xD2`。

### 10.8 HAL 版为什么建议检查返回值？

**答**：因为返回值能区分 I2C 调用是否成功。否则 `id` 可能保持旧值或默认值，LED 现象会误导排查。

## 11. 工程实现步骤

### 11.1 需求分析

本课只做 MPU6050 在线检测和唤醒，不做姿态角计算。先确认 I2C 寄存器读写可靠，后续读取传感器数据才有意义。

### 11.2 硬件核查

确认 MPU6050 供电、GND、PB6/PB7 接线、模块上拉和 AD0 电平。若模块是 5V 供电板，要确认 I2C 电平对 STM32 安全。

### 11.3 寄存器路线

配置 I2C1 后，先写 `0x6B=0x00`，再读 `0x75`。重点观察地址阶段 ACK、寄存器地址写入、重复起始和 RXNE。

### 11.4 HAL 路线

配置 `hi2c1` 后，用 `HAL_I2C_Mem_Write()` 写配置寄存器，用 `HAL_I2C_Mem_Read()` 读身份寄存器。建议补返回值判断。

### 11.5 工程思维

调传感器时先读 WHO_AM_I，不要一上来读数据寄存器。身份寄存器稳定、简单，是区分“通信不通”和“数据处理不对”的第一道门。

传感器调试应该分阶段：第一阶段只验证供电、地址、ACK；第二阶段读 WHO_AM_I；第三阶段写配置寄存器并读回确认；第四阶段再读数据寄存器。直接从姿态数据开始，会把硬件连接、I2C 协议、寄存器配置、数据换算全混在一起。

本课选择 WHO_AM_I 正是为了建立这个工程节奏。读到 `0x68` 只能证明基本通信和身份正确，不代表量程、采样率、滤波、坐标轴方向都配置好了。它是起点，不是完整驱动终点。

### 11.6 常见工程陷阱

地址左移、AD0 电平、寄存器地址长度、上拉电阻、模块供电最容易出错。不要把 `0x75` 当 I2C 地址，它只是 MPU6050 内部寄存器地址。

## 12. 运行现象

MPU6050 接线和地址正确时，LED 快速闪烁。若 ID 不等于 `0x68`，LED 慢速闪烁。

如果程序卡住不闪，可能卡在寄存器版等待 I2C 标志的位置；如果 HAL 版一直慢闪，可能读函数失败但代码没有检查返回值。

## 13. 常见问题排查

### 13.1 LED 慢闪

说明读到的 ID 不是 `0x68`。检查地址、AD0、供电、GND、PB6/PB7、上拉。

### 13.2 程序卡在等待 `ADDR`

地址阶段无 ACK。优先查模块是否在线、地址是否应为 `0x68` 或 `0x69`。

### 13.3 读到 `0x00` 或 `0xFF`

常见于 SDA/SCL 接线、上拉、电平或读流程异常。用逻辑分析仪看是否真的收到 ACK 和数据字节。

### 13.4 HAL 版没有明显错误但 ID 不对

给 `HAL_I2C_Mem_Write()` 和 `HAL_I2C_Mem_Read()` 加返回值判断，并在失败时点亮 LED 或断点查看 `hi2c1.ErrorCode`。

如果返回值是 `HAL_TIMEOUT`，多半是总线或从机响应问题；如果是 `HAL_ERROR`，再看 `hi2c1.ErrorCode` 判断是否 AF 应答失败、总线错误或仲裁相关错误。不要只看 `id`，因为 `id` 是结果变量，不是通信过程证据。

另外确认 HAL 地址参数是否用左移后的 `0xD0`。不同资料有的写 7 位地址 `0x68`，有的写 8 位地址 `0xD0`，移植代码时最容易在这里错一位。

### 13.5 后续读传感器数据不变化

确认已经写 `PWR_MGMT_1=0x00` 唤醒。身份寄存器能读到，不代表采样配置已经完整。

## 14. 本课最核心的结论

1. 访问 I2C 传感器要区分器件地址和内部寄存器地址。
2. MPU6050 常见 7 位地址是 `0x68`，写地址字节是 `0xD0`，读地址字节是 `0xD1`。
3. `WHO_AM_I` 是验证通信链路的优先寄存器。
4. `PWR_MGMT_1=0x00` 用于唤醒传感器。
5. 读寄存器需要先写寄存器地址，再重复起始读数据。
6. HAL 的 `Mem_Read/Mem_Write` 封装了寄存器访问流程，但地址长度和设备地址仍要自己填对。

## 15. 建议你现在怎么读这节课

先把 `0x68/0xD0/0xD1/0x6B/0x75` 五个值分清楚。再对照寄存器版读写函数，看“器件地址”和“寄存器地址”分别在哪一步发送。最后读 HAL 版，确认每个参数对应哪一层含义。

## 16. 扩展练习

1. 给 HAL 版补充返回值检查和错误闪烁模式。
2. 将 AD0 接高，把地址改为 `0x69` 验证地址变化。
3. 读取加速度 X 轴高低字节寄存器，组合成 16 位原始值。
4. 用逻辑分析仪抓取读 `WHO_AM_I` 的完整 I2C 波形。
