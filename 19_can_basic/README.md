# 第 19 课：CAN 基础

## 1. 本课要学什么

前面你已经学过：

- `UART` — 异步串行，2 线
- `SPI` — 同步串行，4 线，一主多从
- `I2C` — 同步串行，2 线，多主多从

这一课进入另一类非常重要的现场总线：**CAN**

CAN 和前面几种通信方式很不一样。它最核心的两个特点：

1. **它是"报文总线"** — 不是字节流，而是完整的报文帧
2. **靠报文 ID 表达含义和优先级** — 不是靠从设备地址选择目标

### CAN vs UART/SPI/I2C 核心区别

| 特性 | UART | SPI | I2C | **CAN** |
|------|------|-----|-----|---------|
| 线数 | 2（TX/RX） | 4（SCK/MOSI/MISO/NSS） | 2（SCL/SDA） | **2（CANH/CANL）** |
| 通信方式 | 异步 | 同步 | 同步 | **异步** |
| 寻址方式 | 无 | 片选线 | 7 位地址 | **报文 ID + 过滤器** |
| 多节点 | 否 | 是（额外 NSS） | 是（共线寻址） | **是（总线仲裁）** |
| 最大距离 | 短 | 短 | 短 | **长（1km+）** |
| 检错 | 无 | 无 | 无 | **CRC + 错误帧** |
| 典型应用 | 调试、GPS | 传感器、Flash | 传感器、EEPROM | **汽车、工业控制** |

## 2. 本课核心概念

### 2.1 报文（Message）

CAN 没有"字节流"的概念。最小传输单位是**一帧报文**，包含：

- **ID**：11 位（标准帧）或 29 位（扩展帧）
- **DLC**：数据长度码（0~8）
- **Data**：0~8 字节数据
- **CRC**：校验和
- **ACK**：应答位

### 2.2 发送邮箱（Tx Mailbox）

F103 的 bxCAN 有 **3 个发送邮箱**（Mailbox 0/1/2）。

每个邮箱存放一帧待发送报文。CPU 先把 ID、DLC、数据装进邮箱，然后置发送请求。

### 2.3 接收 FIFO

收到的报文不会直接跳进变量，而是先进入接收 FIFO。

F103 有 **2 个接收 FIFO**（FIFO0/FIFO1），每个可存 3 帧报文。

### 2.4 过滤器（Filter）

**这是 CAN 独有的概念** — UART/SPI/I2C 都没有。

为什么需要过滤器？在 CAN 总线上，所有节点的报文都被所有节点收到。但大多数节点只关心特定 ID。过滤器在硬件层面筛选——不匹配的报文直接丢弃，CPU 无需参与。

F103 有 **14 个过滤器组**（Filter Bank 0~13），每个可配置为：

- **掩码模式**：用掩码匹配一组 ID
- **列表模式**：匹配特定 ID 列表

### 2.5 内部回环模式（Loopback Mode）

**为什么本课先用内部回环？**

真实 CAN 通信需要：MCU 控制器 + CAN 收发器 + 另一个 CAN 节点 + CANH/CANL 布线 + 两端 120Ω 终端电阻。如果一起上，你很难分清问题是出在控制器配置还是收发器接线。

内部回环模式：
- TX 在控制器内部直接环回到 RX
- 报文经历完整路径：发送邮箱 → 过滤器 → FIFO
- **不依赖外部收发器和总线**

**但这不等于真实 CAN 总线已打通。** 内部回环只验证了控制器配置正确。

## 3. 本课最终 Demo

### 3.1 硬件连接

使用内部回环模式，不需要外接 CAN 收发器和终端电阻。

但需要知道 F103 的默认 CAN 引脚（后续总线实验会用）：

- `PA12` → `CAN_TX`（复用推挽输出）
- `PA11` → `CAN_RX`（浮空输入）

本课会按标准方式初始化这两个引脚。

### 3.2 运行现象

程序每隔约 1 秒：

1. 发送一帧标准帧（ID=0x123, DLC=2, data=0xA5, 0x3C）
2. 从 FIFO0 接收回环报文
3. 校验 ID、DLC、数据是否一致
4. 一致 → LED 翻转；不一致 → LED 常亮

## 4. 寄存器版核心寄存器

### 4.1 MCR（主控制寄存器）

| 位 | 字段 | 含义 |
|----|------|------|
| bit 0 | INRQ | 请求进入初始化模式（1=进入，0=退出） |

很多 CAN 配置（如 BTR、过滤器）只能在初始化模式下修改。

### 4.2 MSR（主状态寄存器）

| 位 | 字段 | 含义 |
|----|------|------|
| bit 0 | INAK | 初始化模式确认（1=已进入） |

### 4.3 BTR（位时序寄存器）

CAN 速率计算公式：

```
每 bit 的 tq 数 = 1 (Sync) + TS1 + TS2
CAN 速率 = PCLK1 / (BRP × tq 总数)
```

本课配置（PCLK1=36MHz）：

| 字段 | 寄存器值 | 实际值 |
|------|---------|--------|
| BRP[9:0] | 3 | 4 分频 |
| TS1[3:0] | 12 | 13 tq |
| TS2[2:0] | 3 | 4 tq |
| SJW[1:0] | 0 | 1 tq |
| **LBKM** | **1** | **内部回环模式** |

计算：tq 总数 = 1 + 13 + 4 = 18，速率 = 36MHz / (4 × 18) = **500kbps**

> 注意：BTR 的寄存器值 = 实际值 - 1。例如 BRP=3 → 实际 4 分频。

### 4.4 发送邮箱寄存器（以邮箱 0 为例）

| 寄存器 | 字段 | 说明 |
|--------|------|------|
| sTxMailBox[0].TIR | STID[10:0] | 标准帧 ID（**左移 21 位**到 bit 31~21） |
| | TXRQ | 发送请求（写 1 启动） |
| sTxMailBox[0].TDTR | DLC[3:0] | 数据长度 |
| sTxMailBox[0].TDLR | 低 4 字节 | data[0]~data[3] |
| sTxMailBox[0].TDHR | 高 4 字节 | data[4]~data[7] |

### 4.5 接收 FIFO 寄存器（以 FIFO0 为例）

| 寄存器 | 说明 |
|--------|------|
| RF0R.FMP0[1:0] | FIFO0 中的报文数（0~3） |
| RF0R.RFOM0 | 释放当前报文（写 1） |
| sFIFOMailBox[0].RIR | 收到的 ID（STID 在 bit 31~21） |
| sFIFOMailBox[0].RDTR | 收到的 DLC |
| sFIFOMailBox[0].RDLR | 收到的低 4 字节 |
| sFIFOMailBox[0].RDHR | 收到的高 4 字节 |

### 4.6 过滤器相关寄存器

| 寄存器 | 说明 |
|--------|------|
| FMR.FINIT | 进入过滤器初始化模式 |
| FA1R.FACT0 | 激活过滤器 0 |
| FS1R.FSC0 | 32 位尺度 |
| FM1R.FBM0 | 0=掩码模式，1=列表模式 |
| FFA1R.FFA0 | 0=分配到 FIFO0 |
| sFilterRegister[0].FR1 | ID 配置 |
| sFilterRegister[0].FR2 | 掩码配置 |

**全放行配置**：FR1=0, FR2=0（掩码全 0 → 任何 ID 都放行）

## 5. 寄存器版初始化流程

```
1. 系统时钟 72MHz
2. 开 CAN1 时钟（CAN1 挂在 APB1）
3. PA11(RX) 浮空输入 + PA12(TX) 复用推挽
4. MCR.INRQ = 1 → 请求进入初始化模式
5. 等待 MSR.INAK = 1（确认已进入）
6. 配置 BTR（位时序 + LBKM 回环模式）
7. 配置全放行过滤器（FR1=0, FR2=0）
8. MCR.INRQ = 0 → 退出初始化模式
9. 等待 MSR.INAK = 0
10. 开始收发
```

---

## 6. 寄存器版发送流程

```
1. 等待 TSR.TME0 = 1（邮箱 0 空闲）
2. 写 TIR = (std_id << 21)（标准帧 ID 左移 21 位）
3. 写 TDTR = len（DLC）
4. 写 TDLR/TDHR = 数据
5. 置 TIR.TXRQ = 1（请求发送）
```

## 7. 寄存器版接收流程

```
1. 等待 RF0R.FMP0 > 0（FIFO0 有报文）
2. 读 RIR → 右移 21 位得 ID
3. 读 RDTR → 取低 4 位得 DLC
4. 读 RDLR/RDHR → 解析数据
5. 写 RF0R.RFOM0 = 1（释放 FIFO 槽位）
```

## 8. HAL 版新增 API

### 8.1 `HAL_CAN_Init()`

根据 `CAN_HandleTypeDef` 初始化 CAN 控制器。内部写入 MCR 和 BTR。

关键配置：
- `Mode = CAN_MODE_LOOPBACK` → 内部回环模式
- `Prescaler = 4` → BRP=4
- `TimeSeg1 = CAN_BS1_13TQ`, `TimeSeg2 = CAN_BS2_4TQ`

### 8.2 `HAL_CAN_ConfigFilter()`

配置过滤器。本课用全放行（ID=0, Mask=0, 掩码模式, 32 位）。

### 8.3 `HAL_CAN_Start()`

启动 CAN 通信。清除 SLEEP 模式和 INRQ 标志。

### 8.4 `HAL_CAN_AddTxMessage()`

添加报文到发送邮箱。参数为 `CAN_TxHeaderTypeDef` 结构体，包含 StdId、DLC、IDE、RTR。

### 8.5 `HAL_CAN_GetRxFifoFillLevel()`

查询 FIFO 中的报文数。对应寄存器版读取 RF0R.FMP0。

### 8.6 `HAL_CAN_GetRxMessage()`

从 FIFO 取回报文。参数为 `CAN_RxHeaderTypeDef` 输出结构体 + 数据缓冲区。

## 9. 本课最容易出错的地方

| 问题 | 后果 |
|------|------|
| 标准帧 ID 没左移 21 位 | 发送的 ID 完全不对 |
| 过滤器没放行 | 发送成功但 FIFO 收不到 |
| 忘了释放 FIFO（RFOM0） | FIFO 满后新报文被丢弃 |
| 位时序乱配 | 回环模式下可能也能工作，但真实总线会出问题 |
| 以为内部回环成功 = 总线打通 | 外部收发器、终端电阻、接线都可能有问题 |

---

## 10. 本课你真正应该掌握什么

1. CAN 是**报文总线**，不是字节流
2. 报文经过**发送邮箱 → 过滤器 → FIFO** 这条链
3. **标准帧 ID 在寄存器中要左移到 bit 31~21**
4. **过滤器**是 CAN 独有的——硬件筛选报文，CPU 无需参与
5. **内部回环模式**适合先学控制器本体
6. HAL 的 CAN API 封装了发送邮箱、FIFO、过滤器的操作
7. 内部回环成功 ≠ 真实 CAN 总线已打通——还需要外接收发器和终端电阻
