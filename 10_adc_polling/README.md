## 5. 先建立一个最基本的脑图

本课整条链路如下：

```
电位器（模拟电压 0~3.3V）
       │
       ▼
  PA1（模拟输入模式）
       │
       ▼
  ADC1 模数转换（12位，0~4095）
       │
       ▼
  等待 EOC 标志
       │
       ▼
  读取 DR 寄存器
       │
       ▼
  比较阈值 → 控制 LED
```

1. 把 `PA1` 配置成模拟输入
2. 打开 ADC1 时钟
3. 给 ADC 配置时钟分频（72MHz / 6 = 12MHz）
4. 选择要采样的通道：通道 1
5. 设置采样时间（239.5 cycles）
6. 打开 ADC，并做校准
7. 软件触发一次转换
8. 等待转换完成（轮询 EOC）
9. 从结果寄存器里读出数字值
10. 根据这个数字值判断 LED 状态

**最重要的理解点是：**

- ADC 不是直接返回电压
- 而是把电压转换成一个数字码值

例如 12 位 ADC 常见结果范围是：

- `0 ~ 4095`

换算公式：

```
电压 = ADC值 × VREF+ / 4095
     = ADC值 × 3.3V / 4095
```

## 6. 先认识本课里出现的核心名词

### 6.1 ADC 是什么

ADC 全称是：

- Analog to Digital Converter

中文一般叫：

- 模数转换器

它的作用就是：

- 把模拟电压转换成数字值

### 6.2 为什么需要 ADC

因为很多真实世界的量本质上是连续变化的，例如：

- 电位器电压
- 光敏电阻电压
- 温度传感器输出电压

CPU 本身不能直接"理解"这种连续模拟电压，它更擅长处理数字。

所以就需要 ADC 来做中间转换。

### 6.3 什么叫通道

ADC 有多个输入通道。

你可以把它理解成：

- ADC 前面挂了很多个可选输入口

本课里我们只用一个：

- `ADC1_IN1`

也就是：

- 通道 1

它对应的 GPIO 引脚就是：

- `PA1`

### 6.4 为什么模拟输入脚要配置成模拟模式

**普通 GPIO 输入模式**的路径：

```
引脚 → 施密特触发器（数字整形）→ 输入数据寄存器
```

施密特触发器会对模拟信号造成畸变，使 ADC 采样不准。

**模拟输入模式**的路径：

```
引脚 → 模拟开关（断开数字通路）→ ADC 采样保持电路
```

模拟模式会：

1. 断开数字输入通路（节省功耗）
2. 让引脚电压直接进入 ADC 采样保持电路
3. 关闭上下拉电阻

### 6.5 什么是采样时间

ADC 在真正开始转换前，并不是瞬间就能准确拿到输入电压。

它要先经过一个"采样保持"过程：

```
采样开关打开 → 采样电容充电到输入电压 → 采样开关关闭 → 开始转换
```

这个过程需要时间，这就是：

- **采样时间**

采样时间由 `SMPR` 寄存器中的 `SMP` 位控制，可选范围 1.5 ~ 239.5 个 ADC 时钟周期。

采样时间太短时，可能会导致：

- 采样电容没充满就开始转换
- 结果偏低且不稳定
- 尤其当信号源输出阻抗较高时更明显

### 6.6 12 位分辨率是什么意思

STM32F103 的 ADC 是 12 位的。

12 位意味着：

- 输入电压范围（0~3.3V）被分成 2^12 = 4096 个台阶
- 每个台阶的电压 = 3.3V / 4096 ≈ 0.8mV
- 这就是"分辨率"——能分辨的最小电压变化

如果换成 10 位 ADC：

- 2^10 = 1024 个台阶
- 分辨率 = 3.3V / 1024 ≈ 3.2mV（更粗糙）

### 6.7 `SMPR` 是什么

`SMPR` 是采样时间寄存器。

它的作用就是：

- 为不同通道设置采样时间

F103 有两个 SMPR 寄存器：
- `SMPR1`：管理通道 10~17
- `SMPR2`：管理通道 0~9

每个通道用 3 位控制采样时间：

| SMP 值 | 采样周期数 |
|--------|-----------|
| 000 | 1.5 cycles |
| 001 | 7.5 cycles |
| 010 | 13.5 cycles |
| 011 | 28.5 cycles |
| 100 | 41.5 cycles |
| 101 | 55.5 cycles |
| 110 | 71.5 cycles |
| 111 | 239.5 cycles |

### 6.8 `SQR` 是什么

`SQR` 是规则组通道序列寄存器。

你可以先把它理解成：

- 规定"这次要采哪些通道、按什么顺序采"

F103 有三个 SQR 寄存器：
- `SQR1`：最高位存规则组长度 L，低位存第 13~16 个通道
- `SQR2`：存第 7~12 个通道
- `SQR3`：存第 1~6 个通道

本课是最简单的情况：

- 规则组长度 L = 0（表示 1 个通道）
- SQ1 = 1（第一个通道是通道 1）

### 6.9 `DR` 是什么

`DR` 是：

- Data Register，数据寄存器

ADC 转换完成后，结果最终就放在这里。

所以本课读取 ADC 的本质就是：

- 等转换结束
- 再读 `ADC1->DR`

注意：读取 `DR` 后，EOC 标志会自动清除（F103 的特性）。

### 6.10 `EOC` 是什么

`EOC` 是：

- End Of Conversion，转换结束标志

当 ADC 完成一次转换后，`SR` 寄存器中的这个标志会置位。

所以轮询方式的核心动作就是：

- 不断检查 `EOC`
- 直到它表示"转换完成"

### 6.11 为什么 F103 ADC 初始化里要做校准

F103 这代 ADC 使用前通常会做：

- 复位校准
- 开始校准

ADC 内部有电容阵列，存在制造偏差。

校准过程会：

1. 复位校准：清除之前的校准值
2. 开始校准：测量内部偏差，存储在内部校准寄存器中

之后每次转换都会自动用这个值修正。

如果不校准，结果可能会有几个 LSB 的偏移。

### 6.12 ADC 时钟为什么不能随便设

ADC 内部也有自己的时钟约束。

在 F103 中，ADC 时钟来自：

- `PCLK2` 再分频

ADC 的最大时钟频率限制：

- 约 14MHz（典型值）

超过这个频率，ADC 内部比较器可能来不及稳定，导致结果不准确。

常见做法是把 ADC 时钟压到一个较稳妥的范围，比如：

- `12MHz`

例如本课会用：

- `PCLK2 = 72MHz`
- ADC 分频 `/6`
- 得到 `12MHz`

### 6.13 HAL 的 ADC 三个常见 API 分别干什么

本课 HAL 版会重点用到：

| HAL 函数 | 对应寄存器操作 | 作用 |
|----------|---------------|------|
| `HAL_ADC_Start()` | `CR2 |= EXTTRIG \| SWSTART` | 启动一次 ADC 转换 |
| `HAL_ADC_PollForConversion()` | `while (!(SR & EOC))` | 等待转换完成（带超时） |
| `HAL_ADC_GetValue()` | `return DR` | 读取转换结果 |

## 7. 本课 Demo 要实现什么

本课最终 Demo 是：

- `PA1` 接电位器中间端
- MCU 轮询采样 `PA1` 电压
- 把采样值和一个阈值比较
- 若采样值大于约一半量程，则点亮 LED
- 否则熄灭 LED

### 7.1 为什么先不做串口打印

因为本课要先把 ADC 的基本采样链路讲透。

如果现在同时引入 UART，会把主线分散。

后面学到 UART 后，再把 ADC 值打印出来会更自然。

### 7.2 ADC 值到电压的换算

假设你读到 ADC 值 = 2048：

```
电压 = 2048 × 3.3V / 4095 ≈ 1.65V
```

如果读到 0：

```
电压 = 0V（接 GND）
```

如果读到 4095：

```
电压 ≈ 3.3V（接 VREF+）
```

## 8. 寄存器版代码逐步讲解

对应代码文件：

- [reg/src/main.c](/home/myself/workspace/mcu/stm32/STM32F103C8T6/10_adc_polling/reg/src/main.c)

### 8.1 寄存器版主线

1. 配系统时钟到 72MHz
2. 配 `PC13` 为输出
3. 配 `PA1` 为模拟输入
4. 打开 ADC1 时钟
5. 设置 ADC 时钟分频（/6 → 12MHz）
6. 配规则组长度和通道（SQR1.L=0, SQR3.SQ1=1）
7. 配采样时间（SMPR2.SMP1=111 → 239.5 cycles）
8. 打开 ADC，复位校准，开始校准
9. 软件触发单次转换（CR2.EXTTRIG + CR2.SWSTART）
10. 轮询 `EOC`（while (!(SR & EOC))）
11. 读取 `DR`
12. 根据结果控制 LED

### 8.2 关键寄存器速查

| 寄存器 | 位/字段 | 本课值 | 含义 |
|--------|---------|--------|------|
| `RCC->APB2ENR` | ADC1EN | 1 | 开启 ADC1 时钟 |
| `RCC->CFGR` | ADCPRE | 10 | PCLK2 / 6 = 12MHz |
| `ADC1->SQR1` | L[3:0] | 0 | 规则组长度 = 1 - 1 = 0 |
| `ADC1->SQR3` | SQ1[4:0] | 1 | 第一个通道 = 通道 1 |
| `ADC1->SMPR2` | SMP1[2:0] | 111 | 采样时间 = 239.5 cycles |
| `ADC1->CR2` | ADON | 1 | ADC 上电 |
| `ADC1->CR2` | RSTCAL | 1→0 | 复位校准 |
| `ADC1->CR2` | CAL | 1→0 | 开始校准 |
| `ADC1->CR2` | EXTTRIG | 1 | 允许触发 |
| `ADC1->CR2` | SWSTART | 1 | 软件启动转换 |
| `ADC1->SR` | EOC | 1 | 转换完成标志 |
| `ADC1->DR` | [11:0] | 结果 | 12 位转换结果 |

## 9. HAL 版代码逐步讲解

对应代码文件：

- [hal/src/main.c](/home/myself/workspace/mcu/stm32/STM32F103C8T6/10_adc_polling/hal/src/main.c)

### 9.1 HAL 版的重点看什么

HAL 会把 ADC 配置拆成：

- ADC 整体初始化（句柄 Init 成员 → `HAL_ADC_Init()`）
- 通道配置（`ADC_ChannelConfTypeDef` → `HAL_ADC_ConfigChannel()`）

所以你会看到：

- `ADC_HandleTypeDef`
- `ADC_ChannelConfTypeDef`

这两个结构体分别表达不同层次的配置。

### 9.2 HAL 版的三步轮询流程

```c
HAL_ADC_Start(&hadc1);               // 1. 启动转换
HAL_ADC_PollForConversion(&hadc1);   // 2. 等待完成
value = HAL_ADC_GetValue(&hadc1);    // 3. 读取结果
```

### 9.3 `HAL_ADC_ConfigChannel()` 在做什么

它负责：

- 指定当前采样哪个通道（写 SQR3 的 SQ1）
- 设置该通道的采样时间（写 SMPR2 的 SMP1）
- 规定该通道在规则组中的顺序

## 10. 寄存器版和 HAL 版对比

### 10.1 寄存器版重点看什么

- 模拟输入模式（MODE=00, CNF=00）
- ADC 时钟分频（ADCPRE）
- 通道选择（SQR3.SQ1）
- 采样时间（SMPR2.SMP1）
- 校准（RSTCAL + CAL）
- `EOC` 和 `DR`

### 10.2 HAL 版重点看什么

- HAL 如何表达 ADC 整体配置
- HAL 如何表达通道配置
- 轮询采样三步：开始、等待、读值

### 10.3 两者如何一一对应

| 功能 | 寄存器版 | HAL 版 |
|------|---------|--------|
| 开 ADC 时钟 | `RCC->APB2ENR |= ADC1EN` | `__HAL_RCC_ADC1_CLK_ENABLE()` |
| ADC 分频 | `RCC->CFGR |= ADCPRE_DIV6` | `__HAL_RCC_ADC_CONFIG(DIV6)` |
| 配置通道 | `SQR3.SQ1 = 1` | `sConfig.Channel = CHANNEL_1` |
| 设采样时间 | `SMPR2.SMP1 = 111` | `sConfig.SamplingTime = 239CYCLES` |
| 校准 | `CAL = 1→0` | `HAL_ADC_Init()` 内部完成 |
| 启动转换 | `CR2 \|= EXTTRIG \| SWSTART` | `HAL_ADC_Start()` |
| 等待完成 | `while (!(SR & EOC))` | `HAL_ADC_PollForConversion()` |
| 读结果 | `return DR` | `HAL_ADC_GetValue()` |

## 12. 常见问题排查

### 12.1 LED 完全没反应

优先检查：

1. 电位器接线是否正确（两端 3.3V + GND，中间 PA1）
2. `PA1` 是否真的配置成模拟输入
3. ADC1 时钟是否打开
4. ADC 是否完成校准（CAL 位是否回到 0）
5. 是否真的等到了 `EOC`（SR.EOC 是否置位）

### 12.2 采样值总是接近 0 或接近满量程

优先检查：

1. 电位器中间端是否真的接在 `PA1`
2. 两端是否分别接 `3.3V` 和 `GND`
3. 通道是否选对为通道 1（SQR3.SQ1 = 1）

### 12.3 数值不稳定

优先检查：

1. 采样时间是否太短（建议用 239.5 cycles）
2. 接线是否松动
3. 模拟输入脚是否错误地配置成普通数字模式
4. 电位器本身质量是否良好（劣质电位器接触不良）

## 13. 本课要点总结

本课最核心的结论如下：

1. **ADC 用于把模拟电压转换成数字值**，12 位分辨率范围 0~4095
2. **电压换算公式**：V = ADC 值 × 3.3V / 4095
3. **`PA1` 对应 `ADC1_IN1`**，查数据手册确认引脚功能
4. **模拟输入脚必须配置成模拟模式**，否则施密特触发器干扰采样
5. **采样时间和通道选择是 ADC 的核心配置**
6. **`EOC` 表示一次转换完成**，`DR` 里存放最终转换结果
7. **F103 的 ADC 初始化里，校准是重要步骤**（RSTCAL → CAL）
8. **ADC 时钟不能超过 ~14MHz**，本课用 72MHz / 6 = 12MHz
9. **轮询方式 CPU 在等待期间不能做别的事**——这是下一课要改进的

### 13.1 一条完整的数据流

```
PA1 模拟电压（0~3.3V）
→ ADC1 采样保持（239.5 cycles）
→ ADC1 逐次逼近转换（12.5 cycles）
→ ADC1->DR 寄存器（12 位结果）
→ CPU 读取 → 比较 → 控制 LED
```

### 13.2 ADC 转换时间计算

在 12MHz 的 ADC 时钟下：

```
总时间 = 采样时间 + 转换时间
       = 239.5 cycles + 12.5 cycles
       = 252 cycles
       = 252 / 12MHz ≈ 21 微秒
```

所以每次读取 ADC，CPU 要等大约 21 微秒。
