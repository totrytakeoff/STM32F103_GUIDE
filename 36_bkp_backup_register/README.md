# 第 36 课：BKP 备份寄存器

## 1. 本课到底在学什么

本课表面现象是：程序检查 `BKP->DR1` 里是否已经保存 `0xA55A`。如果没有，就写入这个标记；之后主循环根据 `DR1` 是否等于 `0xA55A` 控制 PC13 LED 快慢闪。

真正要学的是 STM32F1 的备份域：

```text
普通 RAM：复位后通常不可靠，掉电后丢失
BKP 备份寄存器：位于备份域，某些复位后仍可保持
```

本课完整链路是：

```text
打开 PWR/BKP 时钟
  -> 设置 PWR->CR.DBP 允许写备份域
  -> 读取 BKP->DR1
  -> 如果不是 0xA55A，就写入 0xA55A
  -> 后续复位后再次读取 DR1
  -> 用 LED 快慢闪表示标记是否存在
```

这节课不是讲 RTC 计时，而是先讲 RTC 所在的“备份域”里还有一组可保存小量状态的 BKP 寄存器。

## 2. 本课学习目标

学完本课，你应该能回答：

1. 备份域是什么，和普通外设寄存器有什么不同？
2. `BKP->DR1` 为什么复位后还能保存值？
3. 为什么访问 BKP 前要打开 PWR 和 BKP 两个时钟？
4. `PWR->CR.DBP` 为什么必须置位？
5. `0xA55A` 在本课里代表什么？
6. BKP 寄存器适合保存什么，不适合保存什么？
7. HAL 版 `HAL_PWR_EnableBkUpAccess()` 对应哪个寄存器动作？

- 开发板：STM32F103C8T6 BluePill
- 下载器：ST-Link
- LED：PC13

本课不需要外部模块。若板子接了 VBAT 电池，备份域在主电源断开后也可以保持；若没有 VBAT，普通复位期间通常能观察到保持效果，但完全断电后可能丢失。

## 5. 先建立一个最基本的脑图

```text
system_clock_72mhz_init()
  -> 正常跑主程序

pc13_led_init()
  -> 提供可见反馈

bkp_init()
  -> RCC->APB1ENR 打开 PWR/BKP
  -> PWR->CR.DBP=1 解锁备份域写保护

main()
  -> 读 BKP->DR1
  -> 不等于 0xA55A：写入 0xA55A
  -> 等于 0xA55A：说明标记已存在
  -> LED 快闪表示 DR1 当前正确
```

备份寄存器的价值在于：它能保存“上次已经初始化过”“上次状态是什么”这类很小的状态信息。

## 6. 先认识本课里出现的核心名词

### 6.1 `备份域` 是什么

备份域是 STM32 内部一块特殊区域，包含 RTC 相关寄存器和 BKP 备份寄存器。

它属于芯片电源/保持域，不完全等同于普通 APB 外设。只要备份域供电保持，它里面的值可以跨某些复位保存。

如果备份域被复位或 VBAT/主电源都断掉，里面的值会丢失。

### 6.2 `BKP` 是什么

`BKP` 是 Backup Registers，备份寄存器外设。

它属于备份域。STM32F103 上有多个 `DRx` 数据寄存器，可以保存少量 16 位数据。

本课只用 `DR1`，先把最小读写链路讲清楚。

### 6.3 `BKP->DR1` 是什么

`DR1` 是 BKP 的第 1 个数据寄存器。

它是 16 位寄存器，本课保存 `0xA55A`。程序复位后再次读取，如果值仍在，说明备份域没有被清掉。

如果没解锁备份域写保护，写 `DR1` 可能不会生效。

### 6.4 `PWR` 是什么

`PWR` 是 Power Control，电源控制外设。

它负责低功耗、备份域写保护等电源相关控制。本课用它的 `CR.DBP` 位允许写备份域。

只打开 BKP 时钟还不够，备份域写保护仍会挡住写操作。

### 6.5 `DBP` 是什么

`DBP` 是 Disable Backup domain Protection。

`PWR->CR.DBP=1` 表示关闭备份域写保护，允许软件写 RTC/BKP 相关寄存器。

它属于保护机制层。没有这个保护，程序误写可能破坏 RTC 和备份数据。

### 6.6 `RCC_APB1ENR_PWREN` 是什么

这是 APB1 上 PWR 外设的时钟使能位。

要写 `PWR->CR`，必须先打开 PWR 时钟。否则 DBP 配置无法可靠生效。

### 6.7 `RCC_APB1ENR_BKPEN` 是什么

这是 APB1 上 BKP 外设的时钟使能位。

要访问 `BKP->DR1`，必须打开 BKP 时钟。

### 6.8 `0xA55A` 是什么

`0xA55A` 是本课人为选择的魔数标记。

它不是硬件规定值，只是一个容易识别的 16 位模式。程序用它判断 `DR1` 是否已经被写过。

### 6.9 `VBAT` 是什么

`VBAT` 是备份域电源引脚。

如果主电源断开但 VBAT 仍供电，RTC 和 BKP 备份寄存器可以继续保持。很多 BluePill 未必默认接电池，所以完全断电保持效果取决于板子硬件。

### 6.10 `备份域写保护` 是什么

备份域写保护是芯片防误写机制。它保护 RTC 和 BKP 这类跨复位保存的数据，避免普通程序跑飞后随手改掉后备状态。

本课的写入顺序必须是先打开 PWR 时钟，再设置 `PWR->CR.DBP=1`，最后写 `BKP->DR1`。如果顺序错了，读 `DR1` 可能正常，但写入不一定生效，这会表现为每次复位后都像“第一次运行”。

### 6.11 `HAL_PWR_EnableBkUpAccess()` 是什么

HAL 版用它解锁备份域写保护。

底层对应：

```c
PWR->CR |= PWR_CR_DBP;
```

它不负责写 `BKP->DR1`，只负责打开写权限。

### 6.12 `__HAL_RCC_BKP_CLK_ENABLE()` 是什么

这是 HAL 打开 BKP 时钟的宏。

底层对应 `RCC->APB1ENR` 的 BKPEN 位。

### 6.13 `__HAL_RCC_PWR_CLK_ENABLE()` 是什么

这是 HAL 打开 PWR 时钟的宏。

底层对应 `RCC->APB1ENR` 的 PWREN 位。没有它，`HAL_PWR_EnableBkUpAccess()` 无法正确写 PWR 寄存器。

## 7. 寄存器版代码逐步讲解

### 7.1 系统时钟和 LED

代码先配置 72MHz 时钟，再配置 PC13 输出。BKP 不依赖 72MHz 才能工作，但课程工程保持统一时钟初始化。

### 7.2 `bkp_init()` 打开 PWR/BKP 时钟

```c
RCC->APB1ENR |= RCC_APB1ENR_PWREN | RCC_APB1ENR_BKPEN;
```

PWR 用来解锁备份域，BKP 用来访问备份寄存器。两个时钟都要开。

### 7.3 解锁备份域

```c
PWR->CR |= PWR_CR_DBP;
```

这一步允许写备份域。若没有它，后面的 `BKP->DR1=MAGIC` 可能写不进去。

这一步只解除“写保护”，不是给备份域供电。能不能跨完全断电保持，还要看 VBAT 硬件是否供电。

### 7.4 读取 `BKP->DR1`

```c
if(BKP->DR1!=MAGIC)
```

程序先检查 DR1 当前值。若之前写过且备份域没丢失，应读到 `0xA55A`。

### 7.5 写入标记

```c
BKP->DR1=MAGIC;
```

这把 `0xA55A` 保存到备份寄存器。普通复位后，程序再次运行时可以读回这个标记。

### 7.6 LED 快慢判断

主循环根据 `BKP->DR1==MAGIC` 使用不同延时。当前值正确时快闪；不正确时慢闪。

### 7.7 为什么本课没有擦除操作

BKP 寄存器不像 Flash 页写入，不需要页擦除。它是可直接读写的小寄存器，但容量很小。

### 7.8 什么会清掉 BKP

备份域复位、VBAT/主电源都断电、或者程序主动复位备份域，都可能清掉 `DR1`。

## 8. HAL 版代码逐步讲解

### 8.1 `HAL_Init()` 和时钟

HAL 版先初始化 HAL，再配置系统时钟和 PC13。BKP 操作本身在后续几行完成。

### 8.2 打开 PWR 时钟

```c
__HAL_RCC_PWR_CLK_ENABLE();
```

对应寄存器版 `RCC_APB1ENR_PWREN`。

### 8.3 打开 BKP 时钟

```c
__HAL_RCC_BKP_CLK_ENABLE();
```

对应寄存器版 `RCC_APB1ENR_BKPEN`。

### 8.4 允许备份域访问

```c
HAL_PWR_EnableBkUpAccess();
```

对应设置 `PWR->CR.DBP=1`。

### 8.5 调用顺序为什么不能乱

`HAL_PWR_EnableBkUpAccess()` 需要 PWR 时钟已经打开，访问 `BKP->DR1` 需要 BKP 时钟已经打开。因此 HAL 版也必须先执行 `__HAL_RCC_PWR_CLK_ENABLE()` 和 `__HAL_RCC_BKP_CLK_ENABLE()`。

如果先写 `BKP->DR1` 再解锁，代码能编译，也可能读起来像普通变量赋值，但硬件写保护会让这次保存失败。

### 8.6 直接访问 `BKP->DR1`

HAL 版仍直接读写 `BKP->DR1`。HAL 帮你打开时钟和权限，但本课的数据寄存器读写仍然很直观。

### 8.7 HAL LED 反馈

`HAL_GPIO_TogglePin()` 翻转 PC13，`HAL_Delay()` 根据 `DR1` 是否正确选择快慢。

### 8.8 HAL 和寄存器版的边界

HAL 封装了时钟、GPIO 和备份域访问权限；但备份寄存器本身仍是芯片固定地址上的硬件寄存器。

## 9. 两个版本真正应该怎么学

寄存器版重点看：

```text
PWREN/BKPEN -> DBP -> BKP->DR1
```

HAL 版重点看：

```text
__HAL_RCC_PWR_CLK_ENABLE
__HAL_RCC_BKP_CLK_ENABLE
HAL_PWR_EnableBkUpAccess
BKP->DR1
```

理解这条链路后，下一课 RTC 的备份域解锁就不会突然冒出来。

## 10. 检验问题清单

### 10.1 BKP 寄存器属于普通 RAM 吗？

**答**：不是。BKP 位于备份域，是特殊硬件寄存器，不是普通 SRAM 变量。

### 10.2 为什么要打开 PWR 时钟？

**答**：因为解锁备份域写保护要写 `PWR->CR.DBP`，访问 PWR 寄存器前必须打开 PWR 时钟。

### 10.3 为什么还要打开 BKP 时钟？

**答**：因为 `BKP->DR1` 属于 BKP 外设寄存器组，访问它需要 BKP 时钟。

### 10.4 `DBP=1` 做什么？

**答**：允许写备份域寄存器。没有它，写 BKP/RTC 相关寄存器可能被保护机制拦住。

### 10.5 `0xA55A` 是硬件规定值吗？

**答**：不是。它只是本课选择的标记值，用于判断备份寄存器是否保存过数据。

### 10.6 完全断电后 DR1 一定保持吗？

**答**：不一定。完全断电后是否保持取决于 VBAT 是否继续给备份域供电。

### 10.7 HAL 版为什么仍然写 `BKP->DR1`？

**答**：因为 HAL 只封装访问权限和时钟，备份寄存器数据读写本身可以直接访问寄存器。

### 10.8 BKP 适合保存大量数据吗？

**答**：不适合。BKP 容量很小，适合保存标志、计数、小状态，不适合保存大数据。

## 11. 工程实现步骤

### 11.1 需求分析

本课要验证“复位后仍可读取一个保存标记”。因此选择一个 16 位 magic 值写入 `DR1` 并重复读取。

### 11.2 硬件核查

确认 PC13 LED 可用。若要测试断电保持，需要确认板子 VBAT 是否有电池或后备供电。

### 11.3 寄存器路线

打开 PWR/BKP 时钟，设置 DBP，读 `BKP->DR1`，必要时写入 `0xA55A`。

### 11.4 HAL 路线

使用 HAL RCC 宏打开 PWR/BKP，调用 `HAL_PWR_EnableBkUpAccess()`，再读写 `BKP->DR1`。

### 11.5 工程思维

BKP 适合保存“是否第一次启动”“上次状态”“低功耗唤醒次数”等小状态。不要把它当 EEPROM 或 Flash 用。

### 11.6 常见工程陷阱

忘开 PWR 时钟、忘开 BKP 时钟、忘设 DBP、误以为无 VBAT 也能断电保持、把 BKP 当大容量存储，都会造成理解错误。

还有一个常见误判是只按复位键测试成功，就以为断电也一定保持。复位保持说明备份域没有被复位；断电保持还必须确认 VBAT 供电路径。

## 12. 运行现象

首次运行时如果 `DR1` 不是 `0xA55A`，程序会写入。之后 LED 快闪。按复位键后再次运行，若备份域未被清除，LED 仍快闪。

如果备份域丢失或写入失败，LED 会慢闪。

## 13. 常见问题排查

### 13.1 复位后标记丢失

确认不是断电复位导致 VBAT 也掉电。再检查是否有代码复位备份域。

### 13.2 写入 DR1 不生效

检查 PWR/BKP 时钟是否打开，`PWR->CR.DBP` 是否置位。

### 13.3 完全断电后不保持

检查 VBAT 是否有后备电源。没有 VBAT 时，备份域断电后数据会丢。

### 13.4 HAL 版不能写入

确认调用顺序是先开 PWR/BKP 时钟，再 `HAL_PWR_EnableBkUpAccess()`，最后写 `BKP->DR1`。

### 13.5 LED 快慢与预期不同

用调试器直接观察 `BKP->DR1`，确认它当前是否等于 `0xA55A`。

## 14. 本课最核心的结论

1. BKP 备份寄存器位于备份域，不是普通 RAM。
2. 写备份域前必须打开 PWR/BKP 时钟并设置 DBP。
3. `BKP->DR1` 可以保存少量 16 位状态。
4. 备份域保持能力取决于复位类型和 VBAT 供电。
5. HAL 的备份域访问宏最终仍对应 PWR/BKP 底层寄存器。
6. BKP 是学习 RTC 备份域前的重要铺垫。

## 15. 建议你现在怎么读这节课

先把 `PWREN/BKPEN/DBP/DR1` 四个点连成链路，再用复位键测试 `DR1` 是否保持。然后再考虑断电和 VBAT 场景。
