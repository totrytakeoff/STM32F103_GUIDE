# 第 14 课：UART 轮询收发

## 1. 本课要学什么

前面你已经学过：

- GPIO
- 时钟
- 定时器
- ADC
- DMA

但这些课程大多还是"板子自己在内部运行"。

从这一课开始，我们正式进入：

**MCU 和外部设备交换数据。**

本课要学的是最常见、也最重要的一种串行通信外设：

- `UART`
- 在 STM32 手册里通常会看到 `USART`

这一课我们先不讲中断，也不讲 DMA，只讲最基础的：

**轮询方式的串口发送和接收。**

也就是：

- CPU 主动去看"能不能发"（TXE 标志）
- CPU 主动去看"有没有收到"（RXNE 标志）

这是学习 UART 最扎实的起点。

### USART vs UART

- `UART` = Universal Asynchronous Receiver/Transmitter（异步串口）
- `USART` = Universal Synchronous/Asynchronous Receiver/Transmitter（同步/异步串口）

STM32F103 芯片里外设名字叫 `USART1/2/3`，但我们只使用异步模式，所以口语上说 UART 没问题。

## 2. 本课最终 Demo

### 2.1 硬件连接

使用 `USART1`：

- `PA9`  → `USART1_TX`
- `PA10` → `USART1_RX`

接线方式：

- BluePill `PA9` 接 USB 转串口模块 `RX`
- BluePill `PA10` 接 USB 转串口模块 `TX`
- BluePill `GND` 接 USB 转串口模块 `GND`

注意：

- 串口一定要交叉接（TX ↔ RX）
- 必须共地
- USB 转串口模块必须是 `3.3V TTL` 电平，不能用 RS232

### 2.2 串口参数

- 波特率：`115200`
- 数据位：`8`
- 停止位：`1`
- 校验位：`无`

即 `115200 8N1`。

### 2.3 运行现象

板子上电后，串口助手会先收到欢迎信息。

然后你可以发送下面这些字符：

- 发送 `1` → 点亮 `PC13` 板载 LED
- 发送 `0` → 熄灭 `PC13`
- 发送 `t` → 翻转 LED 状态

每次收到命令后，MCU 都会通过串口回显结果。

## 3. UART 通信最核心的几个概念

### 3.1 TX 和 RX

- `TX` = Transmit，发送
- `RX` = Receive，接收

本课里：`PA9` 负责发，`PA10` 负责收。

### 3.2 波特率（Baud Rate）

波特率 = 每秒传输的符号数（bit/s）。

双方波特率必须一致，否则通信会乱码。本课设为 `115200`。

### 3.3 数据格式（8N1）

- 8 位数据
- N = 无校验（No Parity）
- 1 位停止位

串口传输 1 个字节的物理过程：

```
空闲（高电平）→ 起始位（低电平）→ bit0 → bit1 → ... → bit7 → 停止位（高电平）
```

所以 1 个字节实际发送 10 位：1 起始 + 8 数据 + 1 停止。

传输时间 = 10 / 115200 ≈ 86.8us。

### 3.4 数据寄存器 `DR`

UART 最核心的数据入口/出口：

- 发送时：把要发的字节写进 `DR`
- 接收时：从 `DR` 里把收到的字节读出来

DR 是一个双用途寄存器：写入 = 发送，读取 = 接收。

### 3.5 状态位 `TXE` 和 `RXNE`

这是轮询方式里最重要的两个状态位，位于 SR（Status Register）中。

**TXE（Transmit Data Register Empty）：**

- = 1：DR 是空的，可以写入下一个要发送的字节
- = 0：DR 里还有数据正在等待发送

**RXNE（Read Data Register Not Empty）：**

- = 1：DR 里收到一个新字节，可以去读了
- = 0：还没收到数据

本课的轮询逻辑，本质上就是一直在看：

- `TXE` 是否置位 → 是就可以写下一个字节
- `RXNE` 是否置位 → 是就可以收数据

## 4. 本课数据流总图

发送方向：

```
CPU → USART1->DR → USART1 硬件移位 → PA9(TX) → USB 转串口 → 串口助手
```

接收方向：

```
串口助手 → USB 转串口 → PA10(RX) → USART1 硬件移位 → USART1->DR → CPU
```

LED 控制方向：

```
串口助手发命令 → USART1 接收 → CPU 判断字符 → 控制 PC13
```

## 5. PA9 为什么要配成"复用推挽输出"

这一点和普通 GPIO 输出不同。

| 模式 | 引脚信号来源 | 适用场景 |
|------|------------|---------|
| 通用推挽输出（CNF=00） | GPIO 寄存器（ODR） | 普通 LED、蜂鸣器等 |
| **复用推挽输出（CNF=10）** | **外设（USART/TIM 等）** | **PWM、UART TX 等** |

因为本课里 `PA9` 不再由 GPIO 软件直接输出高低电平，而是要交给 `USART1` 外设控制。所以必须配成"复用推挽输出"。

CRH 寄存器中：

- MODE9 = 11 → 输出模式，50MHz
- CNF9 = 10 → 复用推挽输出

## 6. 为什么 PA10 配成浮空输入

因为 `PA10` 是接收脚，外部 USB 转串口模块把电平送进来，MCU 只需要读。所以配置为浮空输入。

CRH 寄存器中：

- MODE10 = 00 → 输入模式
- CNF10 = 01 → 浮空输入

## 7. BRR 波特率寄存器详解

BRR（Baud Rate Register）是串口初学者最容易疑惑的地方之一："为什么我只是想发串口，还要管 BRR？"

因为 UART 不是无节奏乱发的。串口双方必须按照相同的时间节拍去发送和采样位流。

BRR 的作用就是：告诉串口硬件"按多快的节拍工作"。

### 7.1 计算公式

```
USARTDIV = 时钟频率 / (16 × 目标波特率)
         = PCLK2 / (16 × 115200)
         = 72,000,000 / (16 × 115200)
         = 72,000,000 / 1,843,200
         = 39.0625
```

### 7.2 BRR 编码

```
整数部分 DIV_Mantissa = 39 = 0x27
小数部分 DIV_Fraction = 0.0625 × 16 = 1

BRR = (DIV_Mantissa << 4) | DIV_Fraction
    = (39 << 4) | 1
    = 624 + 1
    = 0x0271
```

### 7.3 验证

```
实际波特率 = 72MHz / (16 × (39 + 1/16))
           = 72MHz / (16 × 39.0625)
           = 72MHz / 625
           ≈ 115200 ✓
```

注意：如果修改了系统时钟（PCLK2 不再是 72MHz），BRR 必须重新计算！

## 9. 轮询发送是怎么工作的

最小发送流程：

1. 等待 `SR.TXE = 1`（上一字节已发完，DR 空了）
2. 把要发的字节写进 `DR`

```c
while ((USART1->SR & USART_SR_TXE) == 0U) { }
USART1->DR = byte;
```

为什么先等 TXE？

- TXE = 1 表示发送数据寄存器空
- 只有这时你再写新字节才安全
- 如果 TXE = 0 时强制写入，会覆盖上一字节

在 115200 下，发送 1 字节耗时约 86.8us，CPU 就在 while 循环里等待。

---

## 10. 轮询接收是怎么工作的

最小接收流程：

1. 检查 `SR.RXNE`
2. 如果 RXNE = 1，说明收到一个新字节
3. 去读 `DR`

```c
if ((USART1->SR & USART_SR_RXNE) != 0U) {
    ch = (uint8_t)USART1->DR;
}
```

为什么这里用"非阻塞"方式（检查一下，有就读，没有就返回）？

- 因为不知道数据何时来
- 如果用 while(!RXNE) 阻塞等，CPU 会被卡住
- 非阻塞方式让 CPU 在没数据时可以去做其他任务

---

## 11. 寄存器版配置顺序

1. 配系统时钟到 72MHz
2. 初始化 PC13 为 LED 输出
3. 开 GPIOA、AFIO、USART1 时钟
4. 配 PA9 为复用推挽输出（USART1_TX）
5. 配 PA10 为浮空输入（USART1_RX）
6. 写 BRR = 0x0271（115200 波特率）
7. 设 CR1 的 TE + RE（发送/接收使能）
8. 设 CR1 的 UE（USART 总使能）
9. 发送欢迎信息
10. 主循环里轮询 RXNE

## 12. HAL 版出现的 API

### 12.1 `HAL_UART_Init()`

根据 `UART_HandleTypeDef` 里的配置初始化串口。

内部自动完成：
- 计算 BRR 并写入（不需要手算！）
- 配置数据位、停止位、校验位
- 使能 TE、RE、UE

### 12.2 `HAL_UART_Transmit()`

轮询方式发送一段数据。内部轮询 TXE，逐字节发送。

```c
HAL_UART_Transmit(&huart1, data_buffer, length, timeout);
```

timeout = HAL_MAX_DELAY 表示一直等到发完。

### 12.3 `HAL_UART_Receive()`

轮询方式接收数据。内部轮询 RXNE。

```c
HAL_UART_Receive(&huart1, &ch, 1, 0);   // 超时 0ms = 非阻塞检查
```

超时设为 0 的含义：立即检查 RXNE，有数据就返回 HAL_OK，没数据立即返回 HAL_TIMEOUT。

### 12.4 寄存器版 vs HAL 版对照

| 功能 | 寄存器版 | HAL 版 |
|------|---------|--------|
| 开时钟 | `RCC->APB2ENR |= USART1EN` | `__HAL_RCC_USART1_CLK_ENABLE()` |
| 设波特率 | `BRR = 0x0271` | `huart1.Init.BaudRate = 115200` |
| 发送 | `while(!TXE); DR=byte;` | `HAL_UART_Transmit()` |
| 接收 | `if(RXNE) ch=DR;` | `HAL_UART_Receive(&h, &ch, 1, 0)` |

## 13. 本课最容易出错的地方

### 13.1 TX/RX 接反

正确接法：`PA9 → 模块RX`，`PA10 → 模块TX`。

### 13.2 没共地

GND 不连，数据参考电平不一致，大概率通信失败。

### 13.3 波特率不一致

串口助手不是 115200 → 乱码。

### 13.4 PA9 没配成复用推挽输出

串口外设即使开了，也发不出去正确波形。

### 13.5 忘记 `\r\n`

只发 `\n` 的话，很多串口助手不会自动回到行首。用 `\r\n` 才能正确换行。

## 14. 本课你真正应该掌握什么

1. UART 最核心的数据口是 `DR`
2. 轮询发送看 `TXE`
3. 轮询接收看 `RXNE`
4. 波特率靠 `BRR` 或 HAL 初始化参数配置
5. `PA9/PA10` 不是随便设成普通 GPIO 就行，TX 要配成复用推挽
6. 串口通信必须 TX ↔ RX 交叉接线 + 共地 + 一致波特率
