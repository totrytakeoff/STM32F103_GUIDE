# STM32 重学学习大纲

## 一、学习目标

这个仓库用于系统性重学 STM32 开发，目标不是只会“调用库函数把功能跑起来”，而是同时掌握：

- 底层原理：理解外设的时钟来源、寄存器配置、数据流向、触发关系和中断机制。
- 工程应用：理解 HAL 的使用方式、初始化流程、回调机制、代码组织方式和常见工程写法。
- 实战能力：每个知识点都能通过一个真实可运行的 Demo 上板验证，而不是停留在理论层面。
- 迁移能力：学完后不仅能写当前芯片的程序，还能更容易迁移到 STM32 其他系列。

本套学习方案采用下面的原则：

- 一个知识点或一个外设，对应一课。
- 每一课都做两个版本：
  - `寄存器版`：用于讲底层原理和最小配置。
  - `HAL版`：用于讲工程应用和常规开发方式。
- 每一课都必须能够在真实板子上烧录运行并验证现象。
- 理论只服务于“能写出代码并跑起来”，不过度空讲。

## 二、板卡规划

### 1. 主学习板：STM32F103C8T6（BluePill）

这块板子作为主线学习平台，原因如下：

- 外设体系经典，资料多，适合打基础。
- 时钟、GPIO、定时器、中断、ADC、UART、SPI、I2C、DMA、看门狗等内容都能覆盖。
- 复杂度适中，适合从零开始建立 STM32 的整体认知。

### 2. 进阶板：STM32F407（DJI RoboMaster C板）

在 F103 基础打牢之后，切到 F407 继续提升，主要用于：

- 学习更复杂的时钟树和更丰富的外设资源。
- 学习更完整的 DMA、多串口、多定时器等进阶内容。
- 学习更接近实际项目的代码结构和多外设协作方式。

### 3. 高性能板：STM32H7（DM-MC02）

H7 不适合作为入门主线，建议放在后期学习，原因如下：

- 时钟域、总线结构、存储层次、Cache 等内容更复杂。
- 如果基础没打牢，容易把问题搞得过于复杂，影响主线节奏。
- 更适合在已经掌握 STM32 常规开发后，用来做架构对比和高性能进阶。

### 4. 当前总策略

整体策略如下：

- 第一阶段主线使用 `STM32F103C8T6`
- 第二阶段进阶使用 `STM32F407`
- 第三阶段再接触 `STM32H7`

## 三、学习原则

- 不使用 CubeMX 生成外设初始化代码作为主学习路径。
- 使用 PlatformIO 做工程管理、编译、烧录和串口监视。
- 讲原理时优先使用 `CMSIS + 寄存器直写`。
- 讲工程应用时使用 `HAL`。
- 每一课都要把“寄存器版”和“HAL版”对照起来讲，而不是分裂成两条完全独立的路线。
- 每个 Demo 都尽量保持最小、直接、可观察、可复现。
- 每做完一课，都要能说清楚“为什么这样配”，而不是只知道“这样写能跑”。

## 四、每一课的固定结构

以后每一课都尽量按同样的结构展开，这样学习节奏稳定，也方便复盘。

### 1. 学习目标

- 本课要解决什么问题
- 学会哪个外设或哪个核心概念
- 完成本课后应该具备什么能力

### 2. 硬件连接

- 使用哪些引脚
- 是否需要外接模块
- 是否需要上拉电阻、串口模块、逻辑分析仪、示波器、CAN 收发器等辅助硬件

### 3. 原理讲解

- 这个外设挂在哪条总线上
- 时钟从哪里来
- 内部的信号或数据是怎么流动的
- 关键寄存器有哪些
- 最小配置步骤是什么

### 4. 寄存器版 Demo

- 给出最小可运行代码
- 逐步解释关键寄存器的设置
- 说明为什么要按这个顺序配置
- 说明如何通过现象或寄存器值验证配置是否正确

### 5. HAL版 Demo

- 给出最小可运行代码
- 解释 HAL 初始化结构体和配置流程
- 说明 HAL 封装掉了哪些底层细节
- 说明 HAL 的常见用法和常见坑

### 6. 两种实现方式对比

- HAL 简化了什么
- 哪些底层细节只有寄存器版更容易看清
- 哪些场景更适合用寄存器，哪些场景更适合用 HAL
- 出问题时该如何从 HAL 回到底层去排查

### 7. 常见故障与排查

- 外设时钟没开
- 引脚模式不对
- 复用功能没选对
- 频率公式算错
- 中断没开或优先级配置错误
- 标志位理解错

### 8. 扩展练习

- 基于本课内容，再做 1 到 2 个小扩展
- 例如：
  - 固定闪烁改成按键控制闪烁
  - 固定占空比改成呼吸灯
  - 串口轮询接收改成中断接收

## 六、阶段学习计划

## 阶段 0：环境与运行基础

### 目标

- 先搞清楚 PlatformIO 提供了什么、不提供什么
- 搞清楚 STM32 程序在 `main()` 之前是怎么跑起来的
- 搭好后续所有课程的统一工程模板

### 学习内容

- PlatformIO 工程结构
- 编译、烧录、串口监视
- 启动文件和中断向量表
- `Reset_Handler`
- `.data` 和 `.bss` 初始化
- CMSIS 设备头文件的作用
- ST-Link 和串口调试的基本使用方法

### 阶段产出

- 一份干净的课程模板工程
- 一个最小可运行的 STM32 工程

## 阶段 1：基础核心能力

### 目标

- 建立后续所有外设学习都依赖的核心基础

### 课程安排

1. `01_gpio_led`
   - GPIO 输出模式
   - 推挽输出、开漏输出
   - BluePill 板载 LED：`PC13`

2. `02_gpio_key`
   - GPIO 输入模式
   - 浮空输入、上拉输入、下拉输入
   - 软件消抖

3. `03_clock_tree`
   - HSI、HSE、PLL
   - SYSCLK、HCLK、PCLK1、PCLK2
   - 为什么 F103 常用 72MHz

4. `05_systick`
   - Cortex-M3 的 SysTick
   - 毫秒节拍
   - 阻塞延时的实现基础

5. `06_timer_base`
   - 定时器时钟
   - PSC、ARR、CNT
   - 更新事件和定时中断

### 阶段要点

- 先搞明白“时钟使能”和“寄存器访问”的关系
- 搞明白总线频率和定时器频率怎么计算
- 搞明白轮询和中断的差别

## 阶段 2：控制与波形输出

### 目标

- 掌握 STM32 如何基于定时器产生时序和响应外部事件

### 课程安排

6. `08_pwm_basic`
   - PWM 输出
   - 频率和占空比计算
   - LED 调光

7. `09_pwm_advanced`
   - 呼吸灯或蜂鸣器
   - 动态更新占空比

8. `10_exti`
   - 外部中断
   - EXTI 线映射
   - NVIC 基础

9. `11_input_capture`
   - 输入捕获
   - 测频率、测脉宽

### 阶段要点

- 定时器不只是用来延时，它还是 PWM、输入捕获、输出比较等功能的基础
- 中断不是“打开开关就行”，必须理解外设端和 NVIC 端是如何配合的

## 阶段 3：模拟量与数据搬运

### 目标

- 掌握模拟量采集和高效数据传输方式

### 课程安排

10. `15_adc_polling`
   - 单通道 ADC
   - 采样时间
   - 转换结果读取

11. `16_adc_interrupt`
   - EOC 中断
   - 定时触发采样

12. `18_dma_basic`
   - DMA 基本原理
   - 外设到内存、内存到外设

13. `20_adc_dma`
   - ADC + DMA
   - 环形模式

### 阶段要点

- ADC 不只是选个通道那么简单，采样时间和 ADC 时钟都很关键
- DMA 是后续做高效采样和通信的基础能力

### 特别说明

- `STM32F103C8T6` 没有片上 `DAC` 外设，所以 DAC 相关内容不能在这颗芯片上原生学习
- 如果后面要学 DAC，可以切到 F407 或使用外接 DAC 芯片

## 阶段 4：串行通信

### 目标

- 建立最常用数字通信协议的实际开发能力

### 课程安排

14. `21_uart_polling`
   - UART 基本收发
   - 波特率生成

15. `22_uart_interrupt`
   - 串口接收中断
   - 简单缓冲设计

16. `23_uart_dma`
   - 串口 DMA 发送或接收
   - 传输完成或空闲中断处理

17. `30_spi_basic`
   - SPI 主模式
   - CPOL、CPHA
   - 环回测试或简单模块通信

18. `26_i2c_basic`
   - 起始、地址、应答、停止
   - EEPROM、OLED 或传感器访问

19. `32_can_basic`
   - CAN 控制器基本工作方式
   - 过滤器配置
   - 报文收发

### 阶段要点

- UART 是后续调试的重要输出通道，必须尽早学扎实
- SPI 和 I2C 都是串行总线，但时序机制和软件模型完全不同
- F103 学 CAN 需要外接 CAN 收发器

## 阶段 5：系统级能力

### 目标

- 从“外设会用”提升到“系统能稳定工作”

### 课程安排

20. `33_nvic_priority`
   - 中断优先级分组
   - 多中断协作

21. `34_watchdog_iwdg`
   - 独立看门狗
   - 喂狗时序

22. `35_watchdog_wwdg`
   - 窗口看门狗
   - 过早/过晚喂狗的区别

23. `37_rtc_basic`
   - RTC 基础
   - 备份域

24. `39_low_power_sleep`
   - 低功耗基础
   - 睡眠与唤醒

### 阶段要点

- 一个外设能跑起来，不代表系统可靠
- 看门狗、优先级、低功耗、RTC 这些内容是“工程能力”的核心部分

## 阶段 6：RTOS 入门与实践

### 目标

- 理解何时需要 RTOS，以及如何组织一个简单的多任务系统

### 板卡建议

- RTOS 可以先在 F103 上做入门
- 如果要做更舒适的多任务实验，建议切到 F407

### 课程安排

25. `44_freertos_intro_porting`
   - 任务创建
   - Tick 和调度基础

26. `50_freertos_queue`
   - 任务间通信
   - 生产者/消费者模型

27. `59_freertos_uart`
   - 串口接收任务
   - 队列或流缓冲区

28. `60_freertos_adc_dma`
   - ADC + DMA + 任务处理

29. `61_small_system_project`
   - 将 GPIO、定时器、UART、ADC 和 RTOS 组合成一个小系统

### 阶段要点

- RTOS 不是为了替代中断，而是在理解中断后进一步组织复杂系统
- 如果连定时器、中断、DMA、UART 都还没吃透，太早上 RTOS 只会增加混乱

## 七、寄存器版与 HAL版的学习重点

同一课的两个版本，侧重点不同，但必须互相对照。

### 1. 寄存器版重点

重点关注：

- 哪个外设的时钟必须先开
- 哪些寄存器必须配置
- 哪些位控制了核心行为
- 为什么配置顺序不能乱
- 出问题时该看哪些寄存器

学习目标：

- 建立这个外设的底层模型
- 以后能更容易迁移到 STM32 其他型号

### 2. HAL版重点

重点关注：

- 如何快速构建一个可维护的工程实现
- HAL 的初始化流程和调用方式
- 回调机制和常见代码结构
- 实际项目中如何更高效地组织代码

学习目标：

- 建立工程开发速度
- 在使用 HAL 时依然保有底层意识

## 八、实战学习规则

- 每一课都必须先上板验证，再进入下一课
- 如果当前课的时钟来源、引脚映射、关键配置还没搞清楚，就不急着往下走
- 学到 UART 后，要尽快建立串口调试输出能力
- 建议同步记录一份学习笔记，至少包含：
  - 引脚分配
  - 定时器频率计算公式
  - 波特率计算方法
  - ADC 相关配置选择
  - 典型 Bug 和排查结论

## 九、建议准备的辅助硬件

- ST-Link
- USB 转串口模块
- 面包板和杜邦线
- LED、按键、电阻
- 电位器（用于 ADC）
- 逻辑分析仪（建议准备，用于 UART/SPI/I2C 调试）
- 示波器（有的话更好）
- CAN 收发器（用于 CAN 学习）

## 十、阶段里程碑

### 里程碑 A：基础控制能力

完成阶段 1 和阶段 2 前半部分后，应具备：

- 配置系统时钟
- 控制 GPIO 输入输出
- 构建延时和周期定时逻辑
- 使用 PWM 和外部中断

### 里程碑 B：采集与通信能力

完成阶段 3 和阶段 4 后，应具备：

- 采集模拟量
- 使用 DMA 搬运数据
- 完成 UART、SPI、I2C 基础通信
- 理解轮询、中断、DMA 三种方式的区别

### 里程碑 C：项目级开发能力

完成阶段 5 和阶段 6 后，应具备：

- 组织一个小型但结构清晰的嵌入式项目
- 正确使用看门狗和中断优先级
- 在裸机或 RTOS 下组织多外设协作
- 从 Demo 级学习过渡到项目级开发

## 十一、后续每一课的执行方式

后续我们每一课都按下面的方式推进：

1. 明确本课目标和硬件连接
2. 用最少但必要的原理讲解支撑代码实现
3. 完成寄存器版并上板验证
4. 完成 HAL版并上板验证
5. 对比两种实现方式
6. 总结公式、关键寄存器和故障点
7. 如果本课稳定，再增加一个小扩展练习
