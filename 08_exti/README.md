# 第 8 课：外部中断 EXTI

## 1. 本课到底在学什么

前面你已经学过：

- GPIO 输入轮询

也就是：

- CPU 在 `while(1)` 里不断去读按键状态
- 如果按下了，再执行对应动作

这类方法能工作，但有一个明显问题：

- CPU 必须一直主动去“问”按键有没有变化

这节课我们要切换到另一种更重要的模式：

- 让外部事件主动“打断”CPU

这就是：

- 外部中断

本课表面上的现象是：

- 每按一次按键，就切换一次 LED 状态

但真正要学的是：

- `EXTI` 是什么
- `AFIO` 为什么会参与 EXTI 映射
- GPIO 引脚和 EXTI 线怎么对应
- 什么叫上升沿/下降沿触发
- `IMR / FTSR / RTSR / PR` 分别是干什么的
- 为什么外设侧和 `NVIC` 侧都要配置
- 为什么中断里必须清挂起标志

## 4. 实验硬件

- STM32F103C8T6 BluePill

### 4.2 下载器

- ST-Link

### 4.3 板载 LED

- `PC13`

多数 BluePill 上：

- `PC13 = 0`：LED 亮
- `PC13 = 1`：LED 灭

### 4.4 外接按键接线

本课仍然使用和第 2 课一致的按键接法：

- `PA0` 接按键一端
- 按键另一端接 `GND`
- `PA0` 使用内部上拉

所以：

- 松开时：`PA0 = 高电平`
- 按下时：`PA0 = 低电平`

这意味着本课最适合的中断触发方式是：

- 下降沿触发

## 5. 先建立一个最基本的脑图

本课的完整链路如下：

1. 先把 `PA0` 配成输入上拉
2. 再把 `PA0` 映射到 `EXTI0`
3. 配置 `EXTI0` 对下降沿敏感
4. 打开 `EXTI0` 中断屏蔽
5. 在 `NVIC` 中允许 `EXTI0_IRQn`
6. 当按键按下时，`PA0` 从高电平变成低电平
7. 下降沿触发 `EXTI0`
8. CPU 进入 `EXTI0_IRQHandler()`
9. 中断里切换 LED 状态
10. 清除 EXTI 挂起标志，等待下一次中断

本课最重要的理解点有两个：

1. `EXTI` 负责“外部事件检测”
2. `NVIC` 负责“把中断真的送进 CPU”

这两层缺一不可。

## 6. 先认识本课里出现的核心名词

### 6.1 EXTI 是什么

`EXTI` 全称可以理解成：

- External Interrupt/Event Controller

中文一般叫：

- 外部中断/事件控制器

你可以先把它理解成：

- 一个专门负责监听外部引脚变化，并在满足条件时发出中断请求的模块

### 6.2 什么是 EXTI line

`EXTI line` 可以理解成：

- EXTI 的一条输入线

例如：

- `EXTI0`
- `EXTI1`
- `EXTI2`

对于 GPIO 引脚来说，常见对应关系是：

- `Px0` -> `EXTI0`
- `Px1` -> `EXTI1`
- `Px2` -> `EXTI2`

所以本课的：

- `PA0`

天然对应：

- `EXTI0`

### 6.3 为什么还要 AFIO

你可能会问：

- 既然 `PA0` 就是 `EXTI0`，为什么还要配 `AFIO`？

原因是：

- `EXTI0` 这条线不只能接 `PA0`
- 理论上也可以接 `PB0`、`PC0`、`PD0` 等

所以芯片需要一个地方明确指定：

- `EXTI0` 现在到底连的是哪一个端口的 0 号引脚

这个映射配置就在：

- `AFIO->EXTICR`

### 6.4 `AFIO->EXTICR` 是什么

这是 AFIO 的外部中断配置寄存器组。

它的作用是：

- 决定某条 EXTI line 来自哪个 GPIO 端口

本课中要配置的是：

- `EXTI0` 来自 `PA0`

#### 寄存器结构拆解：AFIO_EXTICR

AFIO 共有 4 个外部中断配置寄存器（`EXTICR1` 到 `EXTICR4`），每个寄存器负责 4 条 EXTI 线。

你的代码中使用的是 `AFIO->EXTICR[0]`，它对应的就是手册里的  **AFIO_EXTICR1** 。

**AFIO_EXTICR1 结构如下：**

| **位 (Bits)** | **15 ~ 12**                     | **11 ~ 8** | **7 ~ 4** | **3 ~ 0**                                 |
| ------------------- | ------------------------------------- | ---------------- | --------------- | ----------------------------------------------- |
| **分配线**    | EXTI3                                 | EXTI2            | EXTI1           | **EXTI0**                                 |
| **值含义**    | 0000: PAx ``0001: PBx``0010: PCx``... | 同左             | 同左            | **0000: PA0** ``0001: PB0``0010: PC0``... |

再看你提供的这两行代码：

```
/* AFIO_EXTICR1_EXTI0 是一个宏，通常定义为 0x000F (即最低4位全为1) */
AFIO->EXTICR[0] &= ~AFIO_EXTICR1_EXTI0;
```

#### A. 为什么要用 `&= ~` 操作？

这是嵌入式开发中的“先清零，再赋值”原则。

* `AFIO_EXTICR1_EXTI0` 的值是 `0000 0000 0000 1111`（二进制）。
* 取反 `~` 后变成 `1111 1111 1111 0000`。
* 与 `&=` 运算后，强制将寄存器的  **最低 4 位（即负责 EXTI0 的部分）清为 0** ，而保持高位（EXTI1~3）的原有配置不变。

#### B. 为什么清零就代表选择了 PA0？

根据 ST 的硬件手册：

* **0000 (0x0)** 代表 **PA** 端口
* **0001 (0x1)** 代表 **PB** 端口
* **0010 (0x2)** 代表 **PC** 端口

因为你刚好要配置的是  **PA0** ，而 A 端口对应的数值正好是  **0** 。所以当你完成“清零”操作后，实际上就已经完成了“赋值 0”的操作，即成功将 EXTI0 映射到了 PA0。

---

#### 举一反三：如果是 PB0 或 PC0 怎么办？

如果你想把中断改到  **PB0** ，代码应该这样写：


```
// 1. 先清零最低4位 (EXTI0对应的位置)
AFIO->EXTICR[0] &= ~AFIO_EXTICR1_EXTI0; 
// 2. 写入数值 1 (对应 B 端口)
AFIO->EXTICR[0] |= 0x01; 
```

如果你想把中断改到  **PC5** ：

1. **引脚 5** 落在 `EXTICR[1]`（即 EXTICR2）里。
2. **端口 C** 对应的数值是  **2** 。
3. 对应的位区间是第 4~7 位（负责 EXTI5）。


```
AFIO->EXTICR[1] &= ~AFIO_EXTICR2_EXTI5; // 清零 EXTI5 所在的位
AFIO->EXTICR[1] |= (0x02 << 4);         // 在第4位开始写入值 2
```

### 6.5 `EXTI->IMR` 是什么

`IMR` 是：

- Interrupt Mask Register，中断屏蔽寄存器

你可以把它理解成：

- 允许不允许某条 EXTI 线产生中断请求

如果某条线在这里被屏蔽了，那么即使边沿发生了，也不会形成真正的中断请求。

### 6.6 `EXTI->FTSR` 是什么

`FTSR` 是：

- Falling Trigger Selection Register

也就是：

- 下降沿触发选择寄存器

如果把某条线在这里置位，表示：

- 当该线电平发生高到低变化时，就触发 EXTI

### 6.7 `EXTI->RTSR` 是什么

`RTSR` 是：

- Rising Trigger Selection Register

也就是：

- 上升沿触发选择寄存器

如果本课按键接法是“上拉，按下接地”，那么通常不需要上升沿作为主要触发条件。

因为本课更关心的是：

- 按下这一瞬间

而按下对应的是：

- 高 -> 低

也就是下降沿。

### 6.8 `EXTI->PR` 是什么

`PR` 是：

- Pending Register，挂起寄存器

你可以理解成：

- 记录“哪条 EXTI 线已经触发过，但还没被确认处理”

这节课里最关键的一点就是：

- 中断处理后必须清挂起标志

否则就可能出现：

- 中断反复进
- 看起来像程序异常

### 6.9 `EXTI0_IRQHandler()` 是什么

这是：

- `EXTI0` 这条线对应的中断服务函数

当：

- `EXTI0` 触发了
- 且 `NVIC` 允许这个中断

CPU 就会跳进这个函数里。

### 6.10 为什么还要 NVIC

`EXTI` 负责检测事件，但它不等于 CPU 已经真的处理中断。

要让中断真正进入 CPU，还必须：

- 在 `NVIC` 中使能对应中断号

所以和定时器中断一样，这里也有两层：

1. EXTI 自己允许这条线产生中断
2. NVIC 允许 CPU 接收这个 IRQ

### 6.11 HAL 中的 `HAL_GPIO_EXTI_IRQHandler()` 是什么

这是 HAL 提供的 EXTI 通用处理函数。

通常你会在：

- `EXTI0_IRQHandler()`

里面调用它。

它会帮助你：

- 检查挂起状态
- 清除挂起标志
- 调用回调函数

### 6.12 `HAL_GPIO_EXTI_Callback()` 是什么

这是 HAL 的 GPIO 外部中断回调函数。

可以理解成：

- HAL 风格下“真正写业务逻辑”的地方

所以 HAL 的 EXTI 处理链路通常是：

1. 进入 `EXTI0_IRQHandler()`
2. 调用 `HAL_GPIO_EXTI_IRQHandler(GPIO_PIN_0)`
3. HAL 处理标志位
4. HAL 回调 `HAL_GPIO_EXTI_Callback(GPIO_PIN_0)`

### 8.1 寄存器版主线

1. 配系统时钟到 72MHz
2. 配 `PC13` 为输出

### 8.2 为什么一定要清挂起标志

因为 EXTI 和定时器中断一样，事件发生后会留下“我触发过了”的标记。

如果你不清它，CPU 可能会重复响应，导致：

- 看起来像一直在进中断

## 9. HAL版代码逐步讲解

对应代码文件：

- [hal/src/main.c](/home/myself/workspace/mcu/stm32/STM32F103C8T6/08_exti/hal/src/main.c)

### 9.1 HAL 里按键引脚怎么配置成中断模式

HAL 会用 `GPIO_InitTypeDef` 的：

- `Mode`

字段直接描述：

- 是下降沿中断
- 还是上升沿中断
- 还是双边沿中断

例如本课会使用：

- `GPIO_MODE_IT_FALLING`

这表示：

- 下降沿中断模式

### 9.2 HAL 为什么看起来没手动写 AFIO->EXTICR

因为 HAL 在做 GPIO 外部中断初始化时，会帮你把底层相关配置一起做掉。

但这不代表底层不存在。

本质上，它仍然绕不开：

- EXTI line 映射
- 触发边沿
- 挂起标志处理

### 9.3 HAL 里为什么中断函数和回调要分开

因为 HAL 试图把：

- 底层中断清标志流程

和：

- 用户业务逻辑

分开。

这就是为什么：

- `EXTI0_IRQHandler()` 更像入口
- `HAL_GPIO_EXTI_Callback()` 更像业务处理层

## 10. 寄存器版和 HAL版对比

### 10.1 寄存器版重点看什么

- `AFIO->EXTICR` 为什么存在
- `IMR / FTSR / PR` 各自的职责
- 中断后为什么必须清挂起标志

### 10.2 HAL版重点看什么

- `GPIO_MODE_IT_FALLING` 的表达能力
- `HAL_GPIO_EXTI_IRQHandler()` 和回调函数的分层组织

### 10.3 两者如何一一对应

- 寄存器版配 `AFIO/EXTI` <-> HAL 的中断模式 GPIO 初始化
- 寄存器版写 `EXTI0_IRQHandler()` <-> HAL 的 `EXTI0_IRQHandler() + HAL_GPIO_EXTI_IRQHandler()`
- 寄存器版中断里直接改 LED <-> HAL 在回调里改 LED

## 11. 运行现象

如果程序正常，烧录后你会看到：

- 每按一下按键，LED 状态切换一次

这说明：

- 按键的下降沿事件已经成功触发中断

## 12. 常见问题排查

### 12.1 按键按下完全没反应

优先检查：

1. `PA0` 是否真的配置成输入上拉
2. `EXTI0` 是否真的映射到了 `PA0`
3. 是否开启了下降沿触发
4. `IMR` 是否打开
5. `NVIC` 是否打开 `EXTI0_IRQn`

### 12.2 一按按键就连跳很多次

优先检查：

1. 是否正确清除了 `PR` 挂起标志
2. 是否存在机械按键抖动

注意：

- 本课先聚焦中断机制本身
- 机械抖动问题会在实际工程中进一步处理

### 12.3 HAL 版里回调没进

优先检查：

1. `EXTI0_IRQHandler()` 里是否调用了 `HAL_GPIO_EXTI_IRQHandler(GPIO_PIN_0)`
2. 回调函数签名是否写对
3. GPIO 模式是否真的是中断模式

## 13. 本课要点总结

本课最核心的结论如下：

1. 外部中断是让“外部事件主动打断 CPU”
2. `PA0` 对应 `EXTI0`
3. `AFIO->EXTICR` 用来决定 EXTI 线来自哪个端口
4. `IMR` 决定中断是否放行，`FTSR/RTSR` 决定触发边沿
5. 中断处理后必须清 `PR` 挂起标志
6. EXTI 和 NVIC 两层都要配置
7. HAL 版只是把底层过程封装成更统一的接口
