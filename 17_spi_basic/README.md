# 第 17 课：SPI 基础

## 1. 本课要学什么

前面的通信课程里，你已经学过了 UART：

- 只需要 TX / RX 两根线
- 异步通信，双方约定波特率

这一课开始进入另一条非常重要的总线：**SPI**

SPI 和 UART 核心区别：

| 特性 | UART | SPI |
|------|------|-----|
| 时钟线 | 不需要 | **需要 SCK** |
| 通信方式 | 异步（约定波特率） | **同步（主机提供时钟）** |
| 线数 | TX + RX（2 线） | SCK + MOSI + MISO + NSS（4 线） |
| 传输方向 | 全双工 | **全双工** |
| 设备数量 | 点对点 | **一主多从（通过 NSS 片选）** |
| 收发关系 | 发是发，收是收 | **发和收同时发生** |

### SPI 的本质：两个移位寄存器在交换数据

```
主机 SPI                         从机 SPI
┌──────────────┐               ┌──────────────┐
│  移位寄存器   │──MOSI──────→ │  移位寄存器   │
│     TX       │←──MISO───────│     RX       │
└──────────────┘               └──────────────┘
       │                             │
  SCK 时钟（由主机产生）─────────────→│
```

主机每打一拍 SCK 时钟：
- 从 MOSI 移出 1 位（给从机）
- 从 MISO 移入 1 位（收从机数据）

所以 **SPI 的发送和接收永远是同时发生的**，这正是本课最核心的概念。

## 2. 本课最终 Demo

### 2.1 硬件连接

本课采用**回环（loopback）**验证方式——最简单、最干净。

请做一根最关键的跳线：

- `PA7 (SPI1_MOSI)` 连接到 `PA6 (SPI1_MISO)`

其余引脚：

- `PA5` 是 `SPI1_SCK`（主机输出时钟）
- `PA4` (NSS) 本课不接，使用软件 NSS 管理
- `PC13` 用作板载 LED

也就是说，这节最重要的硬件动作就一件事：**用杜邦线把 PA7 和 PA6 连起来。**

### 2.2 运行现象

程序会每隔约 1 秒做一次 SPI 传输，轮流发送两个测试字节：`0xA5` 和 `0x3C`。

由于 MOSI 和 MISO 被接在一起：

- 发出去什么 → 理论上就收回来什么

程序检查：

- 一致 → 翻转 LED（成功）
- 不一致 → LED 常亮（错误）

所以：**能持续闪灯，说明 SPI 回环成功；一直亮说明有问题。**

## 3. SPI 的 4 根线

### 3.1 SCK（Serial Clock）

由主机输出，提供通信的时钟节拍。双方在时钟边沿同步移出/移入数据。

### 3.2 MOSI（Master Out Slave In）

主机输出、从机输入。本课中 `PA7 = SPI1_MOSI`。

### 3.3 MISO（Master In Slave Out）

主机输入、从机输出。本课中 `PA6 = SPI1_MISO`。

### 3.4 NSS（Slave Select）

片选信号，用于主机选择与哪个从机通信。本课用软件管理（SSM=1, SSI=1）。

---

## 4. CPOL 和 CPHA（SPI Mode）

这是 SPI 最经典也最容易混淆的两个概念。

### 4.1 CPOL（Clock Polarity，时钟极性）

决定 **SCK 空闲时是高电平还是低电平**。

- CPOL = 0 → 空闲时 SCK = 低 ← 本课（Mode 0）
- CPOL = 1 → 空闲时 SCK = 高

### 4.2 CPHA（Clock Phase，时钟相位）

决定 **数据在第几个边沿被采样**。

- CPHA = 0 → 第一个边沿采样 ← 本课（Mode 0）
- CPHA = 1 → 第二个边沿采样

### 4.3 四种 SPI Mode

| Mode | CPOL | CPHA | 空闲 SCK | 采样边沿 | 常见外设 |
|------|------|------|---------|---------|---------|
| **0** | 0 | 0 | 低 | **第一个边沿（上升沿）** | 多数 SPI 设备 |
| 1 | 0 | 1 | 低 | 第二个边沿（下降沿） | 部分设备 |
| 2 | 1 | 0 | 高 | 第一个边沿（下降沿） | 部分设备 |
| 3 | 1 | 1 | 高 | 第二个边沿（上升沿） | 部分设备 |

**SPI 双方的 CPOL/CPHA 必须一致。**

---

## 5. 为什么本课用回环（loopback）

目的：先掌握 SPI 主模式怎么配、数据怎么走，再接实际外设。

如果一上来就接 Flash/OLED/传感器，你会同时面对：

- SPI 本身 + 外设协议 + 初始化命令 + 时序要求

学习焦点会被打散。所以本课先**自己发出去、自己收回来**，把 SPI 本体学透。

## 6. 寄存器版核心寄存器

### 6.1 `SPI1->CR1`（控制寄存器 1）

| 位 | 字段 | 本课值 | 含义 |
|----|------|--------|------|
| bit 2 | MSTR | 1 | **主模式** |
| bit 3-5 | BR[2:0] | 010 | **分频 = /8 → 9MHz** |
| bit 1 | CPOL | 0 | 空闲时 SCK 低 |
| bit 0 | CPHA | 0 | 第一个边沿采样 |
| bit 9 | SSM | 1 | 软件 NSS 管理 |
| bit 8 | SSI | 1 | 内部 NSS 有效 |
| bit 6 | SPE | 1 | SPI 总使能 |

### BR 分频速查表（PCLK2=72MHz）

| BR 值 | 分频 | SCK 频率 |
|-------|------|---------|
| 000 | /2 | 36MHz（可能超限） |
| 001 | /4 | 18MHz |
| **010** | **/8** | **9MHz ← 本课** |
| 011 | /16 | 4.5MHz |
| 100 | /32 | 2.25MHz |
| 101 | /64 | 1.125MHz |
| 110 | /128 | 562.5kHz |
| 111 | /256 | 281.25kHz |

### 6.2 `SPI1->SR`（状态寄存器）

| 位 | 标志 | 含义 |
|----|------|------|
| bit 1 | TXE | 发送缓冲区空，可写 DR |
| bit 0 | RXNE | 接收缓冲区非空，可读 DR |
| bit 7 | BSY | SPI 忙（正在通信） |

### 6.3 `SPI1->DR`（数据寄存器）

写入 = 启动发送，读取 = 产生接收。

因为 SPI 收发同步，写入立刻启动时钟输出并移位。

---

## 7. 软件 NSS 的必要性

在没有外部从机时，如果 NSS 不处理，SPI 主模式可能无法正常工作。

配置方法：

- SSM = 1：软件管理 NSS（不使用物理引脚）
- SSI = 1：内部 NSS 信号为有效电平

这样 SPI 硬件认为"片选已有效"，可以正常产生 SCK 时钟。

---

## 8. 为什么写 DR 用 `*(__IO uint8_t *)&SPI1->DR`

SPI1->DR 是 16 位寄存器。但本课是 8 位数据模式：

- 写 16 位：可能向高 8 位写入无效值
- 读 16 位：高 8 位是无效数据

用 8 位指针强制转换，只操作低 8 位：

```c
*(__IO uint8_t *)&SPI1->DR = tx_byte;   // 只写低 8 位
rx_byte = *(__IO uint8_t *)&SPI1->DR;    // 只读低 8 位
```

## 9. HAL 版新增 API

### 9.1 `HAL_SPI_Init()`

根据 `SPI_HandleTypeDef` 的配置初始化 SPI 外设。内部写入 CR1 各配置位并设置 SPE=1。

### 9.2 `HAL_SPI_TransmitReceive()`

**最符合 SPI 本质的 API**：同时发送和接收。

```c
HAL_SPI_TransmitReceive(&hspi1, &tx_buf, &rx_buf, length, timeout);
```

对应寄存器版中的：等待 TXE → 写 DR → 等待 RXNE → 读 DR → 等待 BSY。

## 10. 本课最容易出错的地方

### 10.1 忘了接 PA7 → PA6

MOSI 发出去的东西，MISO 根本收不到。

### 10.2 PA5/PA7 没配成复用推挽输出

如果配成通用推挽输出，SPI 信号无法通过引脚。

### 10.3 没开软件 NSS（SSM + SSI）

主模式下 NSS 不处理，SPI 可能不工作。

### 10.4 忘了等 BSY

RXNE 只表示数据到接收缓冲区，BSY=0 才表示 SPI 总线完全空闲。

---

## 11. 本课你真正应该掌握什么

1. **SPI 是同步串行总线**，时钟由主机提供
2. **SPI 的发送和接收同时发生**——写 DR 的同时也在收数据
3. MOSI / MISO / SCK / NSS 各自的作用
4. CPOL + CPHA 组合 = SPI Mode，双方必须一致
5. 最小主模式配置关键位：MSTR、BR、CPOL、CPHA、SSM、SSI、SPE
6. `HAL_SPI_TransmitReceive()` 封装了 SPI 同步收发的底层流程
