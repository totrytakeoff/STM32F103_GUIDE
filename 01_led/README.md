# 第 1 课：LED 点灯

## 1. 本课到底在学什么

这节课表面上是在“点亮一个 LED”，但真正要学的不是点灯本身，而是 STM32 程序控制一个引脚输出电平的完整链路。

你后面无论学按键、PWM、串口、SPI、I2C、ADC，本质上都离不开下面这些基础概念：

- 芯片里的某个外设默认不是一直工作的，通常要先开时钟
- 外设能工作后，还要配置它的模式
- 模式配置好之后，才能真正读写它对应的数据寄存器
- HAL 只是把这些底层操作封装成了 API，并不是凭空变出功能

所以本课虽然简单，但非常关键。

如果本课你真正搞明白了，那么你以后看到别的外设时，就不会再觉得“怎么突然多了一个寄存器，怎么突然多了一个 API”。

## 2. 本课学习目标

本课完成后，你应该能回答清楚下面这些问题：

- `PC13` 为什么能控制 BluePill 板载 LED
- `RCC` 是什么，它为什么能控制 GPIO 是否工作
- `RCC->APB2ENR` 是什么寄存器，为什么要改它
- `GPIOC->CRH` 是什么寄存器，为什么 `PC13` 要在这里配置
- `GPIOC->BSRR` 和 `GPIOC->BRR` 是干什么的
- `HAL_GPIO_Init()` 这个 API 到底帮你做了什么
- `HAL_GPIO_WritePin()` 为什么能控制引脚输出高低电平
- 为什么很多 BluePill 板子上 `PC13` 输出低电平时 LED 反而亮

### 4.2 下载器

- ST-Link

### 4.3 板载 LED

BluePill 常见板子的板载 LED 接在：

- `PC13`

而且大多数板子是“低电平点亮”：

- `PC13 = 0`：LED 亮
- `PC13 = 1`：LED 灭

这件事很重要，因为你如果按“高电平点亮”的思路理解，就会误以为程序写错了。

## 5. 先建立一个最基本的脑图

控制 `PC13` 输出电平这件事，可以先粗略理解成下面这个流程：

1. 先让 GPIOC 外设“通电工作”
2. 告诉 GPIOC：我要把 `PC13` 配成输出
3. 再往输出寄存器里写 0 或 1
4. 引脚电平变化后，板载 LED 跟着亮灭

这个流程里每一步都对应着具体寄存器或具体 API。

后面你看到寄存器和 API 时，都要把它们放进这条链路里理解，而不是孤立死记。

## 6. 先认识本课里出现的几个核心名词

这部分非常关键。你要先知道“它是谁、它管什么”，后面看代码才不容易断层。

### 6.1 RCC 是什么

`RCC` 全称是：

- Reset and Clock Control

中文通常叫：

- 复位与时钟控制

它负责很多事情，但对当前这节课来说，只需要先记住一件事：

- 它能决定某个外设有没有时钟

而外设没有时钟，通常就不能正常工作。

你可以把它先粗暴理解成一个“外设供电开关管理器”，虽然这不是严格说法，但对入门很有帮助。

### 6.2 `RCC->APB2ENR` 是什么

这是 `RCC` 模块里的一个寄存器，名字拆开看：

- `APB2`：表示 APB2 总线
- `ENR`：Enable Register，使能寄存器

连起来理解就是：

- `APB2 外设时钟使能寄存器`

它的作用就是：

- 控制 APB2 总线上各个外设的时钟开关

在 STM32F103 里，GPIOA、GPIOB、GPIOC 这些 GPIO 外设都挂在 `APB2` 上，所以你要想让 `GPIOC` 工作，就得先改这个寄存器。

### 6.3 `RCC_APB2ENR_IOPCEN` 是什么

这是一个位掩码宏，名字拆开看：

- `RCC_APB2ENR`：目标寄存器是 `RCC->APB2ENR`
- `IOPCEN`：GPIOC 时钟使能位

所以这句：

```c
RCC->APB2ENR |= RCC_APB2ENR_IOPCEN;
```

意思就是：

- 把 `RCC->APB2ENR` 这个寄存器里控制 GPIOC 时钟的那一位置 1
- 置 1 后，GPIOC 外设时钟打开

### 6.4 GPIOC 是什么

`GPIOC` 是一个指向 GPIOC 外设寄存器组的指针。

你看到：

```c
GPIOC->CRH
GPIOC->BSRR
GPIOC->BRR
```

本质上都是在访问 GPIOC 这个外设内部的不同寄存器。

也就是说：

- `GPIOC` 不是一个普通变量
- 它代表的是芯片里 GPIOC 这组硬件寄存器

### 6.5 `GPIOC->CRH` 是什么

`CRH` 全称可以理解成：

- Configuration Register High

它负责配置高 8 个引脚，也就是：

- `Px8 ~ Px15`

因为 `PC13` 落在这个范围内，所以 `PC13` 的模式要在 `GPIOC->CRH` 里配置。

反过来说，如果你要配 `PC5`，那就不该改 `CRH`，而该改 `CRL`。

### 6.6 `GPIO_CRH_MODE13` 和 `GPIO_CRH_CNF13` 是什么

它们是帮助你操作 `PC13` 配置位的掩码宏。

在 F103 里，每个引脚用 4 bit 配置：

- `MODE[1:0]`
- `CNF[1:0]`

对于 `PC13`，这 4 bit 在 `GPIOC->CRH` 里正好对应一段固定位置。

所以：

- `GPIO_CRH_MODE13` 表示 `PC13` 的 `MODE` 那两位
- `GPIO_CRH_CNF13` 表示 `PC13` 的 `CNF` 那两位

### 6.7 `GPIOC->BSRR` 是什么

`BSRR` 全称是：

- Bit Set/Reset Register

它是一个非常常用的 GPIO 输出控制寄存器。

它的特点是：

- 你往某些位写 1，就能把对应引脚置高
- 不需要先读出原值再修改再写回

本课里这句：

```c
GPIOC->BSRR = GPIO_BSRR_BS13;
```

表示：

- 把 `PC13` 输出为高电平

### 6.8 `GPIOC->BRR` 是什么

`BRR` 全称是：

- Bit Reset Register

它专门用来把某个引脚输出拉低。

本课里这句：

```c
GPIOC->BRR = GPIO_BRR_BR13;
```

表示：

- 把 `PC13` 输出为低电平

### 6.9 `HAL_Init()` 是什么

这是 HAL 库最常见的初始化入口函数之一。

它主要做的是 HAL 运行环境的基础初始化，比如：

- 初始化 HAL 内部状态
- 配置系统的基础时间基准

在本课里你暂时可以把它理解成：

- “HAL 工程启动时的基础准备工作”

后面讲 `SysTick` 和 `HAL_Delay()` 时，我们会把它拆开讲得更细。

### 6.10 `GPIO_InitTypeDef` 是什么

这是 HAL 定义的一个结构体，用来描述你想怎么初始化 GPIO。

你给这个结构体填好参数后，再调用：

```c
HAL_GPIO_Init(GPIOC, &gpio);
```

HAL 就会根据结构体里的内容，去帮你配置底层寄存器。

也就是说，这个结构体其实是在表达：

- 你想配哪个引脚
- 你想让它工作在哪种模式
- 你想让它用什么速度
- 是否要上拉/下拉

### 6.11 `HAL_GPIO_Init()` 是什么

这是 HAL 里用于初始化 GPIO 的核心函数。

它的作用就是：

- 根据 `GPIO_InitTypeDef` 中的配置，修改对应 GPIO 寄存器

你可以把它理解成：

- “寄存器配置动作的一个封装入口”

### 6.12 `HAL_GPIO_WritePin()` 是什么

这是 HAL 里用于控制 GPIO 输出电平的函数。

比如：

```c
HAL_GPIO_WritePin(GPIOC, GPIO_PIN_13, GPIO_PIN_RESET);
```

意思就是：

- 把 `GPIOC` 的 `13` 号引脚输出为低电平

而：

```c
HAL_GPIO_WritePin(GPIOC, GPIO_PIN_13, GPIO_PIN_SET);
```

意思就是：

- 把 `GPIOC` 的 `13` 号引脚输出为高电平

### 6.13 `HAL_Delay()` 是什么

这是 HAL 提供的毫秒级阻塞延时函数。

你可以先把它理解成：

- “让 CPU 暂停等待一段时间”

比如：

```c
HAL_Delay(500);
```

表示：

- 延时大约 500ms

为什么这里说“大约”而不是“绝对精确”，以后我们讲到 SysTick 和时钟时会进一步解释。

## 7. 寄存器版代码逐行讲解

对应代码文件：

- [reg/src/main.c](/home/myself/workspace/mcu/stm32/STM32F103C8T6/01_led/reg/src/main.c)

### 7.1 先看完整逻辑

寄存器版的主逻辑只有三件事：

1. 初始化 `PC13`
2. 输出低电平，LED 亮
3. 输出高电平，LED 灭

不停重复，就形成闪烁。

### 7.2 为什么先写一个 `delay()`

如果没有延时，CPU 执行速度太快，人眼根本看不出亮灭变化。

所以这里先用最原始的空转循环做一个粗糙延时。

注意：

- 这个延时不精确
- 它只是为了第一课能看见现象
- 后面我们会用更正规的 `SysTick` 和定时器来替代它

### 7.3 初始化函数 `gpio_pc13_init()` 在干什么

这个函数负责做两件事：

1. 开 GPIOC 时钟
2. 配置 `PC13` 为普通推挽输出

如果少了第一步，第二步基本就没有意义。

### 7.4 为什么要先清零再设置

你会看到这样一类写法：

```c
GPIOC->CRH &= ~(GPIO_CRH_MODE13 | GPIO_CRH_CNF13);
GPIOC->CRH |= GPIO_CRH_MODE13_1;
```

为什么不直接一次写死？

因为寄存器里除了 `PC13` 对应的 4 bit，还有别的位可能在控制其他引脚。

所以正确思路是：

1. 先把 `PC13` 相关位清掉
2. 再只把我们想要的目标位设置进去

这样不会误伤别的位。

### 7.5 为什么初始化后先输出高电平

因为 BluePill 常见板子的 LED 是低电平点亮。

所以：

- 输出高电平 -> LED 灭

程序初始化完先让 LED 处于熄灭状态，比较直观。

## 8. HAL版代码逐行讲解

对应代码文件：

- [hal/src/main.c](/home/myself/workspace/mcu/stm32/STM32F103C8T6/01_led/hal/src/main.c)

### 8.1 HAL 版和寄存器版本质差异是什么

本质差异只有一句话：

- 寄存器版是你亲手改寄存器
- HAL版是你把“配置意图”告诉 HAL，再让 HAL 帮你改寄存器

底层硬件没有变，变的只是写代码的抽象层次。

### 8.2 `HAL_Init()` 为什么在最前面

因为 HAL 的很多功能不是裸写寄存器，而是建立在 HAL 自己的运行机制上的。

比如本课里用到的：

- `HAL_Delay()`

它依赖 HAL 的基础时间基准。

所以在用这些 HAL 功能前，先调用 `HAL_Init()` 是标准动作。

### 8.3 `GPIO_InitTypeDef gpio = {0};` 为什么这样写

这句的意思是：

- 定义一个 GPIO 初始化结构体
- 并把它所有字段先清零

这样做的好处是：

- 避免结构体里残留未初始化的垃圾值
- 后面只填本课真正需要的字段即可

### 8.4 `__HAL_RCC_GPIOC_CLK_ENABLE()` 是什么

这是 HAL 提供的一个宏，用来打开 GPIOC 时钟。

它的本质仍然是在改：

- `RCC->APB2ENR`

也就是说，HAL 并没有绕开底层寄存器，它只是把这一步包装得更好写、更统一。

### 8.5 `gpio.Pin = GPIO_PIN_13;` 是什么意思

表示：

- 我要配置的是 13 号引脚

这里的 `GPIO_PIN_13` 也是 HAL 预定义好的宏。

### 8.6 `gpio.Mode = GPIO_MODE_OUTPUT_PP;` 是什么意思

拆开理解：

- `OUTPUT`：输出模式
- `PP`：Push Pull，推挽输出

也就是：

- 把这个引脚配置成普通推挽输出

### 8.7 `gpio.Pull = GPIO_NOPULL;` 为什么这里写了但好像没用

这个字段表示是否启用上下拉。

在普通输出模式下，这个字段对 F1 的这类 GPIO 配置并不是本课最核心的关注点，但 HAL 的统一结构体里会保留这个字段。

所以你会看到它被一起填上。

这属于 HAL 的“统一接口设计”，不是凭空多出来的无意义代码。

### 8.8 `gpio.Speed = GPIO_SPEED_FREQ_LOW;` 是什么意思

这个字段表示 GPIO 输出速度等级。

你可以先粗略理解成：

- 输出切换能力的配置等级

本课只是点灯，不需要高速翻转，因此用低速配置即可。

### 8.9 `HAL_GPIO_Init(GPIOC, &gpio);` 到底干了什么

这句的意思是：

- 把 `gpio` 结构体中描述的配置应用到 `GPIOC`

具体来说，它最终会去改对应的底层 GPIO 配置寄存器。

对于本课的 `PC13` 来说，它本质上仍然会落到类似 `CRH` 的配置上。

### 8.10 `HAL_GPIO_WritePin()` 为什么能点亮 LED

因为它最终会去控制 GPIO 的输出寄存器，把引脚输出成高电平或低电平。

而板载 LED 正好挂在 `PC13` 上，所以电平变化就会反映成亮灭变化。

## 9. 两个版本真正应该怎么学

### 9.1 先学寄存器版

原因是：

- 你先知道底层到底动了什么
- 后面看 HAL 才不会觉得 API 是“黑盒”

### 9.2 再看 HAL版

原因是：

- 你会发现 HAL 并不是凭空出现
- 它只是把“开时钟、配模式、写电平”这些底层动作包装起来了

### 9.3 正确的心智模型

以后你看到 HAL API 时，脑子里应该自动出现这样的映射：

- `__HAL_RCC_GPIOC_CLK_ENABLE()` -> 本质是开 GPIOC 时钟
- `HAL_GPIO_Init()` -> 本质是配置 GPIO 模式寄存器
- `HAL_GPIO_WritePin()` -> 本质是控制 GPIO 输出电平

这样你就不容易“会调用，但不知道在干嘛”。

## 10. 运行现象

代码正常时：

- 板载 LED 会周期性闪烁

如果现象相反，比如：

- 你觉得“应该亮的时候灭了”

那通常不是程序有大问题，而是：

- 这块板子的 LED 逻辑是低电平有效

## 11. 常见问题排查

### 11.1 为什么烧录成功了但 LED 完全没反应

检查顺序建议如下：

1. 确认板子确实是 BluePill，板载 LED 接的是 `PC13`
2. 确认程序确实下载成功
3. 确认 GPIOC 时钟已经使能
4. 确认你改的是 `CRH` 而不是 `CRL`
5. 确认 `PC13` 被配置成输出
6. 确认 LED 是否为低电平点亮

### 11.2 为什么寄存器版比 HAL版更容易写错

因为寄存器版要求你自己承担下面这些细节：

- 找对寄存器
- 找对位段
- 先清零再设置
- 不误伤其他位

这也是它的学习价值所在。

### 11.3 为什么这节课不用精准毫秒延时

因为这节课核心是 GPIO，不是时间系统。

如果第一课就把时钟树、SysTick、中断、毫秒节拍全引进来，信息量会过大，反而把主线冲散。

所以第一课故意只保留“能观察到现象”的最小延时方案。

## 12. 本课最核心的 6 个结论

1. GPIO 外设要先开时钟，再配置，再使用
2. `GPIOC` 是否工作，先看 `RCC->APB2ENR`
3. `PC13` 属于高 8 位引脚，所以配置在 `GPIOC->CRH`
4. 控制引脚输出电平，可以用 `BSRR/BRR`
5. HAL 并不是魔法，本质仍然是在操作底层寄存器
6. BluePill 板载 LED 通常是低电平点亮
