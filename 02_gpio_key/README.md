# 第 2 课：按键输入控制 LED

## 1. 本课到底在学什么

这节课表面上是在做一件很简单的事：

- 按下按键，板载 LED 亮
- 松开按键，板载 LED 灭

但这节课真正要学的不是“按键控制灯”这个现象，而是 STM32 如何读取一个引脚的输入电平。

你在上一课已经学过了：

- 怎么让 GPIO 作为输出
- 怎么通过输出高低电平控制 LED

这一课则是在上一课的基础上，切换到 GPIO 的另一半能力：

- 输入

也就是说，这节课真正学习的是：

- GPIO 输入模式是怎么配置的
- 为什么输入引脚不能悬空
- 什么是上拉输入
- STM32F103 里“输入上拉/下拉”为什么不仅和 `CNF` 有关，还和 `ODR` 有关
- 怎么从输入数据寄存器里读出某个引脚当前是高电平还是低电平
- HAL 里的输入配置 API 本质上对应了哪些底层动作

如果这一课你真正吃透了，那么后面学：

- 按键中断
- 外部中断 EXTI
- 编码器输入
- 串口接收状态判断
- 各种数字输入信号检测

都会更容易理解。

## 2. 本课学习目标

本课完成后，你应该能清楚回答下面这些问题：

- GPIO 输入模式和输出模式的根本区别是什么
- 为什么按键输入如果不做上拉或下拉，会出现“悬空”
- `GPIOA->IDR` 是什么寄存器，为什么要读它
- `PA0` 为什么配置在 `GPIOA->CRL` 里，而不是 `CRH`
- STM32F103 中“输入上拉/下拉”为什么还要配 `ODR`
- `HAL_GPIO_ReadPin()` 为什么能读到按键状态
- 为什么本课里按键按下时读到的是低电平

## 4. 硬件环境

### 4.1 开发板

- STM32F103C8T6 BluePill

### 4.2 下载器

- ST-Link

### 4.3 板载 LED

- `PC13`

多数 BluePill 上：

- `PC13 = 0`：LED 亮
- `PC13 = 1`：LED 灭

### 4.4 外接按键接线

本课使用一个最常见的接法：

- `PA0` 接按键一端
- 按键另一端接 `GND`

也就是：

- 按键松开时：`PA0` 通过内部上拉保持高电平
- 按键按下时：`PA0` 被按键直接拉到低电平

所以本课中的按键逻辑是：

- 松开：读到高电平
- 按下：读到低电平

这叫做：

- 低电平有效按键

## 5. 先建立一个最基本的脑图

本课整条控制链路如下：

1. 打开 GPIOA 和 GPIOC 的时钟
2. 把 `PA0` 配置成输入上拉
3. 把 `PC13` 配置成输出
4. CPU 反复读取 `PA0` 当前电平
5. 如果读到低电平，说明按键按下
6. 按下则点亮 LED，松开则熄灭 LED

这里最关键的转折点是：

- 输入模式下，不是“往输出寄存器写值”
- 而是“从输入寄存器读取值”

所以这节课第一次引入一个新的关键寄存器：

- `IDR`

## 6. 先认识本课里出现的几个核心名词

### 6.1 输入模式和输出模式的区别是什么

你可以先粗略理解成：

- 输出模式：由 STM32 主动往引脚上输出高低电平
- 输入模式：由 STM32 去读取引脚当前外部给它的电平

上一课是 STM32 控灯。

这一课是 STM32 读按键。

### 6.2 什么叫“悬空”

如果一个输入引脚既没有被外部电路明确拉高，也没有被明确拉低，那么这个引脚电平就可能不稳定。

这种状态通常叫：

- 悬空

悬空输入常见表现是：

- 一会儿读到 1
- 一会儿读到 0
- 很不稳定

所以输入引脚通常需要一个“默认状态”，这就要用到：

- 上拉
- 下拉

### 6.3 什么是上拉输入

上拉输入的意思是：

- 默认通过一个电阻把引脚拉到高电平

这样在按键没有按下时，引脚会稳定在高电平，不会悬空。

当按键按下时，如果按键另一端接地，那么引脚会被拉成低电平。

所以“上拉 + 按键接地”是最经典的输入接法之一。

### 6.4 `GPIOA->IDR` 是什么

`IDR` 全称是：

- Input Data Register

中文可以理解成：

- 输入数据寄存器

它的作用就是：

- 反映当前 GPIO 端口各个引脚实际读到的输入电平

例如：

- 如果 `PA0` 当前是高电平，那么 `GPIOA->IDR` 的第 0 位就是 1
- 如果 `PA0` 当前是低电平，那么 `GPIOA->IDR` 的第 0 位就是 0

所以本课里读取按键状态的本质，就是读这个寄存器里的对应位。

### 6.5 `GPIOA->CRL` 是什么

上一课你已经见过 `CRH` 了。

在 STM32F103 中：

- `CRL`：配置引脚 0 到 7
- `CRH`：配置引脚 8 到 15

因为本课的按键接在：

- `PA0`

所以它属于 0 号引脚，要配在：

- `GPIOA->CRL`

### 6.6 输入上拉在 F103 里为什么不只是改 `CNF`

这是本课最重要的新知识点之一。

在 STM32F103 里，把 GPIO 配成“输入上拉/下拉”时，寄存器逻辑不是只改一个地方。

首先，要把该引脚配置成：

- 输入模式
- 上拉/下拉输入类型

也就是：

- `MODE = 00`
- `CNF = 10`

但这还不够。

因为对于 F103：

- 到底是“上拉”还是“下拉”
- 还要看该引脚对应的 `ODR` 位

规则是：

- `ODR = 1`：输入上拉
- `ODR = 0`：输入下拉

也就是说，F103 的输入上拉/下拉不是单靠 `CRL/CRH` 就能完全决定的，还要结合 `ODR`。

这一点非常值得记住，因为它是 F1 这一代 GPIO 的一个典型特征。

### 6.7 `GPIOA->ODR` 在输入模式下为什么还会用到

你可能会奇怪：

- 不是输入模式吗，为什么还要碰 `ODR` 这个看起来像输出寄存器的东西？

原因就是上一条提到的：

- 在 F103 的“输入上拉/下拉模式”里，`ODR` 的对应位被拿来决定内部是上拉还是下拉

所以在本课寄存器版中，你会看到：

- 把 `PA0` 配成输入上拉/下拉模式
- 然后再把 `ODR0` 置 1

这两步合起来，才真正得到：

- 输入上拉

### 6.8 为什么代码里写的是 `BSRR`，却说是在改 `ODR`

这正是 F103 GPIO 初学时最容易糊涂的地方。

先把关系说清楚：

- `ODR` 是输出数据寄存器
- `BSRR` 是置位/复位寄存器

它们不是同一个寄存器，但：

- 往 `BSRR` 写入某一位
- 会改变 `ODR` 中对应位的结果

例如：

```c
GPIOA->BSRR = GPIO_BSRR_BS0;
```

这句话的效果就是：

- 把 `ODR0` 置为 1

也可以近似理解成：

```c
GPIOA->ODR |= GPIO_ODR_ODR0;
```

只是 `BSRR` 更适合做“只改某一位”的操作，因为：

- 不需要先读原值
- 不需要手动改 bit
- 不需要再写回整个寄存器

所以本课里虽然代码没直接写 `ODR`，但它确实是在“通过 BSRR 间接把 ODR0 置 1”。

### 6.9 `GPIOA->CRL |= GPIO_CRL_CNF0_1;` 到底在干什么

这句的目的不是“直接设成上拉”，而是：

- 先把 `PA0` 放进“输入上拉/下拉模式”这个大类

原因是 F103 的输入上拉/下拉要分成两步：

1. 在 `CRL/CRH` 里把引脚模式设置成：
   - `MODE = 00`
   - `CNF = 10`
2. 再通过 `ODR` 对应位决定：
   - `1` 是上拉
   - `0` 是下拉

而：

```c
GPIOA->CRL |= GPIO_CRL_CNF0_1;
```

正是在做第 1 步里的：

- `CNF0 = 10`

因为前面已经先清零了：

```c
GPIOA->CRL &= ~(GPIO_CRL_MODE0 | GPIO_CRL_CNF0);
```

所以这两句连起来看，最终结果是：

- `MODE0 = 00`
- `CNF0 = 10`

也就是：

- 输入上拉

### 6.10 `GPIOA->BSRR = GPIO_BSRR_BS0;` 到底在干什么

这句是在做第 2 步：

- 把 `ODR0` 置为 1

而在 F103 的“输入上拉/下拉模式”里：

- `ODR0 = 1` 表示上拉
- `ODR0 = 0` 表示下拉

所以这句的真正含义不是“输出高电平”，而是：

- 在当前这种特殊输入模式下，告诉芯片内部对 `PA0` 使用上拉

所以你应该把这两句连起来理解：

```c
GPIOA->CRL |= GPIO_CRL_CNF0_1;
GPIOA->BSRR = GPIO_BSRR_BS0;
```

第一句的意思是：

- 进入“输入上拉/下拉模式”

第二句的意思是：

- 在这个模式下选择“上拉”

两句缺一不可。

### 6.11 `HAL_GPIO_ReadPin()` 是什么

这是 HAL 提供的 GPIO 输入读取函数。

它的作用就是：

- 读取某个 GPIO 引脚当前的电平状态

比如：

```c
HAL_GPIO_ReadPin(GPIOA, GPIO_PIN_0)
```

意思就是：

- 读取 `PA0` 当前是高电平还是低电平

本质上，它最终还是会去读取底层的输入数据寄存器。

## 7. 寄存器版代码逐步讲解

对应代码文件：

- [reg/src/main.c](/home/myself/workspace/mcu/stm32/STM32F103C8T6/02_gpio_key/reg/src/main.c)

### 7.1 本课寄存器版要完成什么

寄存器版主逻辑非常直接：

1. 初始化 LED 输出引脚 `PC13`
2. 初始化按键输入引脚 `PA0`
3. 在死循环中读取 `PA0`
4. 如果按下按键，则点亮 LED
5. 如果松开按键，则熄灭 LED

### 7.2 LED 初始化为什么还要保留

因为按键只是输入来源，本课最终还是要通过 LED 呈现结果。

所以必须同时具备：

- 一个输入引脚
- 一个输出引脚

这样才能形成完整的“输入 -> 判断 -> 输出”闭环。

### 7.3 `PA0` 的 4 bit 应该怎么配

对于输入上拉/下拉模式，F103 的目标配置是：

- `MODE0 = 00`：输入模式
- `CNF0 = 10`：上拉/下拉输入

组合起来就是：

- `1000`

注意这里不要和上一课的输出模式混淆。

输入模式时，`MODE` 必须为 `00`。

### 7.4 为什么读按键时要和“按下为低电平”对应起来

因为本课接法是：

- `PA0` 通过内部上拉默认为高电平
- 按键按下时直接接地

所以：

- 读到 0 -> 按下
- 读到 1 -> 松开

也就是说，本课的判断逻辑不是“高电平表示按下”，而是正好相反。

## 8. HAL版代码逐步讲解

对应代码文件：

- [hal/src/main.c](/home/myself/workspace/mcu/stm32/STM32F103C8T6/02_gpio_key/hal/src/main.c)

### 8.1 HAL 版对按键输入是怎么描述的

HAL 中我们会通过 `GPIO_InitTypeDef` 来描述按键引脚：

- 哪个引脚
- 工作在什么模式
- 是否启用上拉

对于本课的 `PA0`：

- 模式是普通输入
- 上拉选择为 `GPIO_PULLUP`

HAL 会把这些配置转换成底层寄存器设置。

### 8.2 `GPIO_MODE_INPUT` 是什么

这表示：

- 普通输入模式

也就是这个引脚不主动输出，而是用于读取外部电平。

### 8.3 `GPIO_PULLUP` 是什么

表示：

- 对该输入引脚启用上拉

本质上，在 F1 这类芯片上，HAL 最终仍然会配置到底层对应的输入上拉/下拉模式和 `ODR` 位。

所以 HAL 只是把“输入上拉”这个意图表达得更直观了。

### 8.4 `HAL_GPIO_ReadPin()` 返回值表示什么

它通常返回：

- `GPIO_PIN_SET`
- `GPIO_PIN_RESET`

在本课中：

- 返回 `GPIO_PIN_RESET` -> 表示 `PA0` 当前是低电平 -> 按键按下
- 返回 `GPIO_PIN_SET` -> 表示 `PA0` 当前是高电平 -> 按键松开

## 9. 寄存器版和 HAL版对比

### 9.1 寄存器版重点看什么

- `PA0` 为什么在 `CRL`
- 输入上拉/下拉模式为什么要配 `CNF`
- 为什么 `ODR0 = 1` 才能形成上拉
- 为什么读输入要看 `IDR`

### 9.2 HAL版重点看什么

- `GPIO_MODE_INPUT` 和 `GPIO_PULLUP` 的工程写法
- `HAL_GPIO_ReadPin()` 的使用方式
- HAL 是如何把“我要一个输入上拉按键”封装成标准接口的

### 9.3 两者如何一一对应

- 寄存器版开时钟 <-> HAL 的 `__HAL_RCC_GPIOx_CLK_ENABLE()`
- 寄存器版改 `CRL/CRH` <-> HAL 的 `HAL_GPIO_Init()`
- 寄存器版读 `IDR` <-> HAL 的 `HAL_GPIO_ReadPin()`
- 寄存器版写 `BSRR/BRR` <-> HAL 的 `HAL_GPIO_WritePin()`

## 10. 运行现象

如果程序正常，烧录后应该看到：

- 按键松开时，LED 熄灭
- 按键按下时，LED 点亮

如果你按下按键后现象相反，比如：

- 按下灭，松开亮

通常不是程序整体错了，而是：

- 接线方式和文档假设不同
- 你实际做成了另一种逻辑极性

## 11. 常见问题排查

### 11.1 按键状态乱跳

优先检查：

- 是否真的启用了上拉
- 按键接线是否正确
- 输入引脚是否悬空

### 11.2 按下按键没反应

优先检查：

1. `PA0` 是否真的接到了按键
2. 按键另一端是否接地
3. GPIOA 时钟是否打开
4. `PA0` 是否被配成输入上拉
5. 是否正确读取了 `IDR` 或使用了 `HAL_GPIO_ReadPin()`

### 11.3 LED 一直亮或一直灭

优先检查：

1. `PC13` 是否仍然正确配置为输出
2. BluePill 板载 LED 是否低电平点亮
3. 按键极性判断是否写反

### 11.4 为什么本课还没加入消抖

因为本课核心是：

- 输入模式
- 上拉
- 电平读取

机械按键抖动确实存在，但“消抖”应该作为下一层知识单独展开，不应该干扰本课主线。

## 12. 本课要点总结

本课最核心的结论如下：

1. GPIO 不仅能输出，也能输入
2. 输入引脚不能悬空，通常要配上拉或下拉
3. `PA0` 属于低 8 位引脚，所以配置在 `GPIOA->CRL`
4. 读取输入电平要看 `IDR`
5. STM32F103 的输入上拉/下拉不仅看 `CNF`，还要看 `ODR`
6. HAL 的输入读取本质上仍然是在访问底层 GPIO 输入状态
